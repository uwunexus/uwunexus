# Part 03: අපේ තාක්ෂණික මෙවලම් (Tech Stack)

පසුගිය පාඩම් දෙකෙන් අපි වෙබ්සයිට් එකක් වැඩ කරන හැටි සහ අපේ UWU-NEXUS Project එක ගැන ඉගෙන ගත්තා. මේ Project එක හදන්න අපි පාවිච්චි කරලා තියෙන්නේ මොන වගේ ආයුධද (Tools) කියලා අපි දැන් බලමු.

Software Engineer කෙනෙක් මේ Tools සෙට් එකට කියන්නේ **Tech Stack** එක කියලා.

අපේ Tech Stack එක ප්‍රධාන කොටස් 3කින් සමන්විතයි:
1. **Next.js (React)** - Frontend එක සඳහා
2. **PHP** - Backend එක සඳහා
3. **MySQL** - Database එක සඳහා

---

## 1. Next.js (Frontend යෝධයා)

මුල් කාලයේ වෙබ්සයිට් හැදුවේ තනිකරම HTML පාවිච්චි කරලා. ඒත් ලොකු වෙබ්සයිට් එකක් හදද්දී, හැම තැනටම එකම කෝඩ් එක (උදාහරණයක් විදිහට ඉහළින් තියෙන Navigation Bar එක) හැම පිටුවකම ලියන්න ගියාම ලොකු කරදරයක් වුණා.

මේකට විසඳුමක් විදිහට Facebook ආයතනයෙන් **React** කියලා අලුත් තාක්ෂණයක් හැදුවා. ඒකෙන් පුළුවන් වුණා කෝඩ් එක පොඩි පොඩි කෑලි (Components) විදිහට කඩලා ආයෙ ආයෙ පාවිච්චි කරන්න (උදා: ලෙගෝ බ්ලොක්ස් එකතු කරලා ගෙවල් හදනවා වගේ).

**Next.js** කියන්නේ මේ React තාක්ෂණය තවත් දියුණු කරලා හදපු, ලෝකයේ දැන් ගොඩක්ම ජනප්‍රිය Frontend Framework එකක්.
* **UWU-NEXUS එකේ පාවිච්චිය:** ඔයා දකින හැම ලස්සන බොත්තමක්ම, හැම Page එකක්ම හදලා තියෙන්නේ Next.js වලින්. 

## 2. PHP (පැරණි ඒත් ශක්තිමත් මොළය)

PHP කියන්නේ ලෝකයේ වෙබ් අඩවි 70% කටත් වඩා ප්‍රමාණයක් අදටත් දුවන Backend භාෂාවයි. Facebook එක මුලින්ම හැදුවෙත් PHP වලින්. 

Next.js වගේ අලුත් තාක්ෂණයක් Frontend එකට පාවිච්චි කරද්දී, ඇයි අපි Backend එකට පරණ PHP ම පාවිච්චි කරන්නේ?
හේතුව තමයි PHP කියන්නේ ඉගෙනගන්න හරිම ලේසි, Database එක්ක කතා කරන්න හරිම පහසු භාෂාවක් වීමයි. 

* **UWU-NEXUS එකේ පාවිච්චිය:** කෙනෙක් ලොග් වෙනකොට පාස්වර්ඩ් එක හරිද කියලා බලන්නේ, සල්ලි ගෙව්වම ඒක කන්ෆර්ම් කරන්නේ මේ PHP කෝඩ් එකෙන්.

## 3. MySQL (පිළිවෙළකට තියෙන ගබඩාව)

අපිට දත්ත (Data) ගබඩා කරන්න ක්‍රම ගොඩක් තියෙනවා. ඒත් ලෝකයේ වැඩියෙන්ම පාවිච්චි වෙන්නේ **Relational Databases (සම්බන්ධතා ඇති දත්ත ගබඩා)** කියන ජාතියයි. MySQL කියන්නේ එහෙම එකක්.

මේක හරියටම Microsoft Excel වගේ. ඒකෙ තියෙනවා වගු (Tables).
උදාහරණයක් විදිහට අපේ Database එකේ තියෙනවා:
* `users` කියලා Table එකක් (මේකේ තියෙන්නේ ළමයින්ගේ නම්, ඊමේල්)
* `events` කියලා Table එකක් (මේකේ තියෙන්නේ ඉවෙන්ට් වල නම්, දින)

මේ Tables දෙක අතර සම්බන්ධතා හදන්න පුළුවන් (උදා: මේ ඉවෙන්ට් එක දැම්මේ මේ ළමයා කියලා).

## මේ තුන කතා කරන්නේ කොහොමද? (The API)

ඔයාට දැන් ප්‍රශ්නයක් එන්න පුළුවන්, **"කොහොමද මේ Next.js (React) කෝඩ් එකයි, PHP කෝඩ් එකයි එකිනෙකාට කතා කරන්නේ? ඒ දෙක භාෂා දෙකක් නේද?"** කියලා.

අන්න ඒකට අපි පාවිච්චි කරනවා **API (Application Programming Interface)** කියලා පාලමක්. 
මේ පාලම හරහා කතා කරන්න එයාලා පාවිච්චි කරන්නේ පොදු භාෂාවක්. ඒකට කියන්නේ **JSON** කියලා. JSON කියන්නේ හරියට ඉංග්‍රීසි භාෂාව වගේ - ලෝකේ ඕනෙම කෙනෙක්ට තේරෙනවා.

1. **Next.js** අහනවා: "මට අලුත් ඉවෙන්ට් ටික දෙන්න" (මේ ඉල්ලීමට කියන්නේ Request එකක් කියලා).
2. **PHP** ගිහින් **MySQL** ගෙන් ඉවෙන්ට් ටික ඉල්ලගෙන, ඒ ටික **JSON** විදිහට ලස්සනට පැකට් කරලා Next.js එකට දෙනවා (මේකට කියන්නේ Response එකක් කියලා).
3. ඊටපස්සේ Next.js ඒක ලස්සනට Screen එකේ පෙන්නනවා.

> **සාරාංශය:**
> * UI එක (පෙනුම) හදන්නේ ලෙගෝ බ්ලොක්ස් වගේ කෑලි එකතු කරන **Next.js** වලින්.
> * මොළය විදිහට තීරණ ගන්නේ **PHP** වලින්.
> * එක්සෙල් ශීට් වගේ දත්ත ගබඩා කරන්නේ **MySQL** වලින්.
> * මේ ඔක්කොම කතා කරන්නේ **API** කියන පාලමෙන් ගමන් කරන **JSON** කියන භාෂාවෙන්.

දැන් ඔයාට පදනම සම්පූර්ණයෙන්ම තේරිලා ඉවරයි. ඊළඟට එන දෙවෙනි මොඩියුලෙන් (Module 2) අපි අත් දෙක ටිකක් කිලුටු කරගෙන ඇත්තටම PHP කෝඩ් එකක් ලියන්නේ කොහොමද කියලා ඉගෙන ගනිමු!
