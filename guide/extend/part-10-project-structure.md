# Part 10: Project එකේ ෆෝල්ඩර සැකැස්ම (File Structure)

දැන් අපි යන්නේ අපේ UWU-NEXUS Project එක ඇතුළේ ඇවිදලා බලන්නයි. ඔයා Project ෆෝල්ඩරය (uwunexus) VS Code වගේ සොෆ්ට්වෙයාර් එකකින් ඕපන් කළාම, වම් පැත්තේ ෆෝල්ඩර ගොඩක් පේන්න තියෙනවා. මුලින්ම දැක්කම මේක මහා සංකීර්ණයි වගේ පෙනුණට, මේක ඇතුළේ තියෙන්නේ හරිම සරල රටාවක්.

ප්‍රධාන වශයෙන්ම අපිට වැදගත් වෙන්නේ ෆෝල්ඩර දෙකක් විතරයි:
1. **`app/`** (අපේ Frontend එක / Next.js)
2. **`backend/`** (අපේ Backend එක / PHP)

---

## 1. `app/` ෆෝල්ඩරය (ඔබට පෙනෙන දේවල්)

මේ ෆෝල්ඩරය ඇතුළේ තමයි වෙබ්සයිට් එකේ ඔයා දකින හැම Page එකක්ම තියෙන්නේ. Next.js වල තියෙන ලස්සනම දේ තමයි, **මේක ඇතුළේ තියෙන ෆෝල්ඩර වල නම් වලින්ම තමයි වෙබ්සයිට් එකේ ලින්ක් (URLs) හැදෙන්නේ.** 

ඔයා `app/` එක ඇතුළට ගියොත් මෙන්න මේ වගේ ෆෝල්ඩර දකින්න පුළුවන්:

* **`app/login/page.tsx`** 
  මේක තමයි ලොග් වෙන පිටුව. කවුරුහරි බ්‍රව්සර් එකේ `uwunexus.com/login` (නැත්නම් `localhost:3000/login`) කියලා ගැහුවොත්, පේන්නේ මේ ෆයිල් එක ඇතුළේ තියෙන කෝඩ් එකයි.
* **`app/signup/page.tsx`**
  මේක ලියාපදිංචි වෙන පිටුව. ලින්ක් එක වෙන්නේ `/signup`.
* **`app/marketplace/page.tsx`**
  මේක තමයි බඩු විකුණන තැන. ලින්ක් එක වෙන්නේ `/marketplace`.

> **මතක තියාගන්න:** ඔයාට අලුතින් Page එකක් වෙබ්සයිට් එකට දාන්න ඕනේ නම් (උදා: `/about` කියලා), ඔයාට තියෙන්නේ `app/` එක ඇතුළේ `about` කියලා අලුත් ෆෝල්ඩරයක් හදලා, ඒක ඇතුළේ `page.tsx` කියලා ෆයිල් එකක් හදන එක විතරයි!

### `app/components/` (නැවත නැවත පාවිච්චි කරන කෑලි)
`app/` එක ඇතුළේ තව විශේෂ ෆෝල්ඩරයක් තියෙනවා `components` කියලා. මේක ඇතුළේ තියෙන්නේ ලෙගෝ කෑලි වගේ පොඩි පොඩි කෝඩ් කෑලි. 
උදාහරණයක් විදිහට, වෙබ්සයිට් එකේ උඩින්ම තියෙන "Navigation Bar" එක හැම පිටුවකම පේනවා නේද? අපි ඒක හැම Page එකේම ආයෙ ආයෙ ලියන්නේ නැහැ. අපි ඒක `Navbar.tsx` කියලා මේ components ෆෝල්ඩරය ඇතුළේ ලියලා, අනිත් පිටු වලට ඒක සම්බන්ද කරනවා.

---

## 2. `backend/` ෆෝල්ඩරය (මොළය)

මේක ඇතුළේ තමයි අපේ PHP කෝඩ් ඔක්කොම තියෙන්නේ. මේ ෆෝල්ඩරය ඇතුළට ගියාම ඔයාට දකින්න පුළුවන් `.php` වලින් ඉවර වෙන ෆයිල් ගොඩක්. 

මේ හැම ෆයිල් එකකින්ම කරන්නේ එක නිශ්චිත කාර්යයක් විතරයි (API Endpoints):

* **`login.php`** - `app/login/page.tsx` එකෙන් ඊමේල් එකයි පාස්වර්ඩ් එකයි එව්වම, ඒක අරන් Database එකෙන් චෙක් කරන්නේ මේකෙන්.
* **`signup.php`** - අලුත් ළමයෙක්ගේ විස්තර Database එකට සේව් කරන්නේ මේකෙන්.
* **`get_events.php`** - Database එකේ තියෙන ඉවෙන්ට් ඔක්කොම අරන් Frontend එකට යවන්නේ මේකෙන්.
* **`db.php`** - මේක තමයි ගොඩක්ම වැදගත් කෙනා. Database එකට Connect වෙන කෝඩ් එක (OOP පාවිච්චි කරලා) තියෙන්නේ මේකේ. අනිත් හැම PHP ෆයිල් එකක්ම මුලින්ම මේ `db.php` එකට කතා කරනවා.

---

## සම්පූර්ණ පින්තූරය (The Big Picture)

කෙනෙක් අපේ වෙබ්සයිට් එක පාවිච්චි කරද්දී මේ ෆෝල්ඩර දෙක කතා කරගන්නේ මෙහෙමයි:

1. ළමයෙක් බ්‍රව්සර් එකේ යනවා `localhost:3000/events` වලට.
2. Next.js වලින් `app/events/page.tsx` ෆයිල් එක ලෝඩ් කරනවා.
3. ඒ ෆයිල් එක ඇතුළේ තියෙන කෝඩ් එකෙන් ඉල්ලීමක් (Request) යවනවා `backend/get_events.php` ෆයිල් එකට.
4. `get_events.php` එක `db.php` හරහා ගිහින් Database එකෙන් ඉවෙන්ට් ටික අරන් එනවා.
5. අරන් ආපු ඉවෙන්ට් ටික (JSON විදිහට) ආපහු `app/events/page.tsx` එකට දෙනවා.
6. Frontend එකෙන් ඒක ලස්සනට පින්තූරත් එක්ක Screen එකේ පෙන්නනවා!

> **සාරාංශය:**
> * පෙනුම වෙනස් කරන්න ඕනේ නම් යන්න ඕනේ **`app/`** ෆෝල්ඩරයට.
> * වැඩ කරන ලොජික් එක වෙනස් කරන්න ඕනේ නම් යන්න ඕනේ **`backend/`** ෆෝල්ඩරයට.

දැන් ඔයා අපේ ගේ (Project එක) ඇතුළේ කාමර තියෙන්නේ කොහෙද කියලා දන්නවා. ඊළඟ පාඩමෙන් අපි ගෙදර තියෙන සේප්පුව (Database) ඇරලා බලමු, ඒකෙ ඇතුළේ දේවල් අසුරලා තියෙන්නේ කොහොමද කියලා. (Database Schema).
