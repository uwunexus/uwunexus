# Part 08: Object-Oriented Programming (OOP) යනු කුමක්ද?

කලින් පාඩම් වලදී අපි Variables (පෙට්ටි) හදලා ඒවට විස්තර දැම්මා. Functions හදලා ඒවට කෝඩ් කෑලි පැකට් කළා. 

හැබැයි මේ විදිහට එක එක තැන Variables 100කුයි, Functions 50කුයි විසිවෙලා තියෙද්දී ඒක හොයාගන්න ලොකු අමාරුවක් වෙනවා නේද? අන්න ඒ වගේ අවුල් සහගත තත්ත්වයන් මඟහරින්න ලෝකයේ මෘදුකාංග ඉංජිනේරුවන් හොයාගත්ත ලස්සන "කෝඩ් ලියන රටාවක්" තමයි **OOP (Object-Oriented Programming)** කියන්නේ.

OOP කියන්නේ අලුත් භාෂාවක් නෙවෙයි. ඒක PHP, Java, Python වගේ ගොඩක් භාෂා වලින් **"අපි කෝඩ් එක පිළිවෙළට ලියන විදිහක්"** විතරයි.

මේක තේරුම් ගන්න අපි එදිනෙදා ජීවිතයේ උදාහරණයක් ගමු. 

---

## 1. Class එකක් කියන්නේ මොකක්ද? (The Blueprint)

හිතන්න ටොයෝටා (Toyota) ෆැක්ටරියෙන් කාර් එකක් හදන්න යනවා කියලා. ඒකට මුලින්ම ඉංජිනේරුවෙක් **"සැලැස්මක් (Blueprint)"** අඳිනවා. මේ සැලැස්ම ඇතුළේ ලියලා තියෙනවා:
* කාර් එකට රෝද 4ක් තියෙන්න ඕනේ.
* කාර් එකට පාටක් තියෙන්න ඕනේ.
* කාර් එකට ඉදිරියට යන්න (Drive) පුළුවන් වෙන්න ඕනේ.
* කාර් එකට නවතින්න (Stop) පුළුවන් වෙන්න ඕනේ.

මේ සැලැස්මෙන් (කාර් එක අඳපු කොළයෙන්) අපිට යන්න බැහැ. ඒක සැලැස්මක් විතරයි.
පරිගණක කේතනයේදී මේ **සැලැස්මට (Blueprint) අපි කියනවා `Class` එකක්** කියලා.

## 2. Object එකක් කියන්නේ මොකක්ද? (The Real Thing)

අර ඉංජිනේරුවා ඇඳපු සැලැස්ම (Class එක) පාවිච්චි කරලා ෆැක්ටරියෙන් ඇත්තටම කාර් 100ක් හදනවා. 
* නිලේෂ් ගන්නවා නිල් පාට කාර් එකක්.
* අමල් ගන්නවා රතු පාට කාර් එකක්.

මේ ඇත්තටම පාරේ යන්න පුළුවන්, අපිට අල්ලන්න පුළුවන් **සැබෑ කාර් එකට අපි කියනවා `Object` එකක්** කියලා. එක සැලැස්මෙන් (Class) අපිට ඕනෙතරම් Objects හදාගන්න පුළුවන්.

## 3. Properties සහ Methods (විස්තර සහ ක්‍රියා)

* **Properties:** අර කාර් එකේ තියෙන විස්තර (පාට, රෝද ගාණ) වලට අපි කියනවා Properties කියලා. (සාමාන්‍ය PHP වලදී අපි මේවට කිව්වේ Variables කියලයි).
* **Methods:** අර කාර් එකට කරන්න පුළුවන් වැඩ (Drive, Stop) වලට අපි කියනවා Methods කියලා. (සාමාන්‍ය PHP වලදී අපි මේවට කිව්වේ Functions කියලයි).

---

## PHP වලින් සරල Class එකක් ලියමු!

දැන් අපි ඉගෙන ගත්ත දේ කෝඩ් එකකින් බලමු. අපි `User` (පරිශීලකයා) කියලා Class එකක් හදමු.

```php
<?php

// 1. මේක තමයි අපේ සැලැස්ම (Class එක)
class User {
    
    // මේවා Properties (විස්තර)
    public $name;
    public $email;

    // මේක Method එකක් (කරන්න පුළුවන් වැඩක්)
    public function sayHello() {
        echo "ආයුබෝවන්, මගේ නම " . $this->name; 
        // $this->name කියන්නේ "මේ Object එකේ නම" කියන එකයි.
    }
}

// ----------------------------------------------------

// 2. දැන් අපි සැලැස්මෙන් ඇත්ත Objects හදමු! (new කියන වචනෙන්)
$student1 = new User();
$student1->name = "Nilesh";      // student1 ගේ නම දැම්මා
$student1->email = "ni@uwu.lk";

$student2 = new User();
$student2->name = "Kamal";       // student2 ගේ නම දැම්මා
$student2->email = "ka@uwu.lk";

// 3. දැන් අපි ඒ ගොල්ලන්ට කතා කරන්න කියමු (Method එකට කතා කරමු)
$student1->sayHello(); // ප්‍රතිඵලය: ආයුබෝවන්, මගේ නම Nilesh
echo "<br>";
$student2->sayHello(); // ප්‍රතිඵලය: ආයුබෝවන්, මගේ නම Kamal

?>
```

**මෙතනදී වෙලා තියෙන දේ බලන්න:**
* අපි `User` කියලා එක සැලැස්මක් හැදුවා. ඒකෙ ඇතුළේ නම සහ ඊමේල් එක දාන්න පුළුවන් කියලා කිව්වා.
* ඊටපස්සේ අපි `$student1` සහ `$student2` කියලා ඇත්තම Objects දෙකක් හැදුවා (`new User()` පාවිච්චි කරලා).
* මේ දෙන්නගේ නම එකිනෙකට පැටලෙන්නේ නැහැ! මොකද මේවා සම්පූර්ණයෙන්ම වෙනම Objects දෙකක්.

> **සාරාංශය:**
> * **OOP** කියන්නේ කෝඩ් ලියන පිළිවෙළකටයි.
> * **Class** කියන්නේ අපි හදන සැලැස්මක්. 
> * **Object** කියන්නේ ඒ සැලැස්මෙන් හැදුන ඇත්තම දේ (`new` පාවිච්චි කරලා හදන).
> * Class එකක් ඇතුළේ තියෙන Variables වලට **Properties** කියනවා.
> * Class එකක් ඇතුළේ තියෙන Functions වලට **Methods** කියනවා.

මේක තේරුම් ගන්න පුළුවන් නම් ඔයා Software Engineering වල ගොඩක් ඉස්සරහින් ඉන්නවා. ඊළඟ පාඩමෙන් අපි බලමු මේ OOP සංකල්පය අපේ UWU-NEXUS එක ඇතුළේ කොහොමද පාවිච්චි කරලා තියෙන්නේ කියලා! (මොකද Database එකට කනෙක්ට් වෙන්නෙත් මේ OOP විදිහටමයි).
