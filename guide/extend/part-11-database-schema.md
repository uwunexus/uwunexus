# Part 11: UWU-NEXUS Database එක තේරුම් ගනිමු (Database Schema)

කලින් පාඩමෙන් අපි බැලුවා Project එකේ ෆෝල්ඩර තියෙන්නේ කොහොමද කියලා. මේ පාඩමෙන් අපි යන්නේ අපේ ගෙදර තියෙන සේප්පුව (Database එක) ඇරලා බලන්න. 

අපේ Project එකේ Database එකේ නම **`uwunexus`**. මේක ඇතුළේ වෙනස් ජාතියේ දේවල් දාන්න වෙන වෙනම Tables (Excel Sheets වගේ) ගොඩක් හදලා තියෙනවා. අපි මේකෙ තියෙන ප්‍රධානම Tables 3ක් සහ ඒවා එකිනෙකට සම්බන්ධ වෙලා (Relationships) තියෙන්නේ කොහොමද කියලා බලමු.

---

## 1. `users` Table (පරිශීලකයින්)

මේක තමයි පදනම. කැම්පස් එකේ ළමයෙක් රෙජිස්ටර් වුණාම එයාගේ විස්තර සේව් වෙන්නේ මේ Table එකේ.

* **id:** අංක 1, 2, 3... (මේක තමයි Primary Key එක. අඳුරගන්න පාවිච්චි කරන අංකය).
* **first_name:** ළමයාගේ මුල් නම.
* **last_name:** ළමයාගේ අග නම.
* **email:** ළමයාගේ ඊමේල් එක.
* **password:** පාස්වර්ඩ් එක (හැබැයි මේක ලියන්නේ අනිත් අයට කියවන්න බැරි වෙන විදිහට කේත කරලයි - Encrypted).
* **uwu_role:** මේ ළමයා සාමාන්‍ය ළමයෙක්ද (user) නැත්නම් ඇඩ්මින් කෙනෙක්ද (admin/superadmin) කියලා තීරණය කරන්නේ මේ Column එකෙන්.

## 2. `events` Table (උත්සව)

කවුරුහරි ඉවෙන්ට් එකක් (උදාහරණයක් විදිහට "Acoustica 2026") හැදුවම ඒක සේව් වෙන්නේ මේකෙයි.

* **id:** ඉවෙන්ට් එකේ අංකය. (Primary Key)
* **title:** ඉවෙන්ට් එකේ නම (Acoustica 2026).
* **event_date / event_time:** තියෙන දිනය සහ වෙලාව.
* **location:** තියෙන තැන (උදා: Main Hall).
* **created_by:** **මේක ගොඩක් වැදගත්!** මේකේ සේව් වෙන්නේ ඉවෙන්ට් එක හදපු ළමයාගේ නම නෙවෙයි, එයාගේ `users` Table එකේ තියෙන **`id`** එකයි (උදා: 5).

## 3. `marketplace_items` Table (වෙළඳපොළ)

කවුරුහරි පොතක් හරි පරණ ලැප්ටොප් එකක් හරි විකුණන්න දැම්මොත් ඒක එන්නේ මේකට.

* **id:** විකුණන භාණ්ඩයේ අංකය.
* **title:** භාණ්ඩයේ නම (උදා: Engineering Math Book).
* **price:** මිල (උදා: 1500.00).
* **seller_id:** විකුණන ළමයාගේ `users` Table එකේ **`id`** එක.

## Tables අතර තියෙන සම්බන්ධතා (Foreign Keys)

ඔයාට දැන් ප්‍රශ්නයක් එන්න පුළුවන්: **"ඇයි ඉවෙන්ට් එක හදපු කෙනාගේ නම හරි ඊමේල් එක හරි කෙලින්ම `events` Table එකේ සේව් කරන්නේ නැතුව, `users` Table එකේ තියෙන `id` අංකය විතරක් සේව් කරන්නේ?"** කියලා.

හිතන්න අපි නම සේව් කළා කියලා. පස්සේ ළමයා එයාගේ නම "Nimal" ඉඳන් "Kamal" කියලා වෙනස් කළොත්, එයා දාපු ඉවෙන්ට් 100කම ගිහින් නම වෙනස් කරන්න වෙනවා! 
ඒත් අපි `id` එක විතරක් දැම්මම (ඒකට කියන්නේ **Foreign Key** කියලා), `id` එක කවදාවත් වෙනස් වෙන්නේ නැති නිසා, නම වෙනස් කළත් ඉවෙන්ට් වලට කිසිම බලපෑමක් වෙන්නේ නැහැ.

මේකට තමයි **Relational Databases (සම්බන්ධතා ඇති දත්ත ගබඩා)** කියලා කියන්නේ.

* `events` Table එකේ `created_by` කියන Column එකෙන් ඇඟිල්ල දික් කරගෙන ඉන්නේ `users` Table එකේ `id` එකටයි.
* `marketplace_items` Table එකේ `seller_id` කියන Column එකෙනුත් ඇඟිල්ල දික් කරගෙන ඉන්නේ `users` Table එකේ `id` එකටයි.

අපේ Backend (PHP) කෝඩ් එකෙන් Database එකට කතා කරද්දී (`SELECT` කරද්දී), මේ Tables දෙකක් එකට සම්බන්ධ කරලා (JOIN කියලා කමාන්ඩ් එකක් පාවිච්චි කරලා) තමයි දත්ත අරන් එන්නේ. එතකොට ඉවෙන්ට් එකේ විස්තරයි, ඒක දාපු කෙනාගේ නමයි දෙකම එකපාර ගන්න පුළුවන්.

> **සාරාංශය:**
> * අපේ Database එකේ ප්‍රධාන දේවල් වලට වෙන වෙනම Tables තියෙනවා (`users`, `events`, `marketplace_items`).
> * ඒ Tables එකිනෙකට සම්බන්ධ වෙලා තියෙන්නේ **`id`** (Primary Key / Foreign Key) කියන කවදාවත් වෙනස් වෙන්නේ නැති අංක හරහායි.

දැන් ඔයාට අපේ Database එක සම්පූර්ණයෙන්ම තේරෙනවා. ඊළඟ පාඩමෙන් අපි බලමු අපේ PHP මොළය (`backend/db.php`) මේ Database ගබඩාවට හරියටම Connect වෙන්නේ කොහොමද කියලා.
