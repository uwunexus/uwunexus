# Part 13: React Components කියන්නේ මොනවාද? (Next.js Intro)

අපි දැන් යන්නේ Backend (PHP) ලෝකෙන් එළියට ඇවිත්, අපේ ඇහැට පේන ලස්සන ලෝකෙට, ඒ කියන්නේ Frontend එකට. අපේ UWU-NEXUS Project එකේ Frontend එක හදන්න අපි පාවිච්චි කරලා තියෙන්නේ **Next.js** කියන තාක්ෂණය.

Next.js කියන්නේ **React** කියන තාක්ෂණය තවත් දියුණු කරලා හදපු එකක්. ඒක නිසා Next.js තේරුම් ගන්න නම්, අපි මුලින්ම React කියන්නේ මොකක්ද කියලා තේරුම් ගන්න ඕනේ.

---

## 1. පැරණි HTML ක්‍රමය vs නවීන React ක්‍රමය

ඉස්සර වෙබ්සයිට් හැදුවේ තනිකරම HTML පාවිච්චි කරලයි. හිතන්න අපිට වෙබ්සයිට් එකේ පිටු 10ක් තියෙනවා කියලා. ඒ හැම පිටුවකම උඩින්ම තියෙනවා එකම විදිහේ Menu Bar (Navigation) එකක්. 

* **පැරණි ක්‍රමය (HTML):** ඔයාට අර Menu Bar එකේ කෝඩ් එක පිටු 10යේම ගිහින් ආයේ ආයේ ලියන්න වෙනවා. දවසක Menu Bar එකේ පොඩි වචනයක් වෙනස් කරන්න ඕනේ වුණොත්, පිටු 10ටම ගිහින් ඒක වෙනස් කරන්න ඕනේ. මේක මහා වදයක්!

* **නවීන ක්‍රමය (React):** Facebook ආයතනයෙන් මේකට විසඳුමක් ගෙනාවා. ඒගොල්ලෝ කිව්වා "අපි මේ කෝඩ් එක පොඩි පොඩි කෑලි වලට කඩමු" කියලා. අන්න ඒ වගේ **නැවත නැවත පාවිච්චි කරන්න පුළුවන් කෝඩ් කෑල්ලකට අපි කියනවා `Component` එකක් කියලා.**

## 2. Component එකක් කියන්නේ හරියට Lego Block එකක් වගේ!

ඔයා පොඩි කාලේ ලෙගෝ (Lego) වලින් සෙල්ලම් කරලා ඇති. එක එක පාට, එක එක හැඩයේ ලෙගෝ කෑලි එකතු කරලා තමයි අපි ලොකු ගෙයක් හදන්නේ. 

React වලදීත් අපි කරන්නේ ඒකමයි.
* අපි වෙනම හදනවා `Button` කියලා පොඩි Component එකක්.
* තව එකක් හදනවා `Navbar` (උඩින් තියෙන මෙනු එක) කියලා.
* තව එකක් හදනවා `Footer` (යටින් තියෙන එක) කියලා.

ඊටපස්සේ අපි මේ කෑලි 3 එකට එකතු කරලා සම්පූර්ණ Page එකක් (ගෙයක්) හදනවා. අන්න ඒ Page එකත් තවත් ලොකු Component එකක්!

## 3. UWU-NEXUS එකේ Component එකක් ලියන්නේ කොහොමද?

React Components ලියන්නේ අමුතු භාෂාවකින් නෙවෙයි, අපි දන්න සාමාන්‍ය **Functions (JavaScript/TypeScript)** පාවිච්චි කරලයි. 

අපේ Project එකේ `app/lost-and-found/page.tsx` වගේ ඕනෙම ෆයිල් එකක් ඇරලා බැලුවොත්, ඔයාට මෙන්න මේ වගේ රටාවක් දකින්න පුළුවන්:

```tsx
// 1. මේක තමයි අපේ Component එක (ප්‍රධාන Function එක)
export default function LostAndFoundPage() {
    
    // 2. මේක ඇතුළේ ලියන්නේ මේ පිටුවට අදාළ ලොජික් එක (උදා: දත්ත ගෙනීම)
    const pageTitle = "නැතිවූ සහ හමුවූ දේවල්";

    // 3. return එකෙන් කරන්නේ අර ලෙගෝ කෑලි ටික (HTML වගේ තියෙන කෝඩ් එක) තිරයට යවන එකයි.
    return (
        <div>
            <h1>{pageTitle}</h1>
            <p>ඔබේ නැතිවූ දේවල් මෙතනින් සොයාගන්න.</p>
        </div>
    );
}
```

මෙතනදී වැදගත්ම දේවල් ටික බලමු:
* **`export default function`**: මේකෙන් කියන්නේ "මේක තමයි මේ ෆයිල් එකේ තියෙන ප්‍රධාන ලෙගෝ කෑල්ල (Component), මේක අරගෙන තිරයේ පෙන්නන්න" කියන එකයි.
* **`return ( ... )`**: මේක ඇතුළේ තමයි ඔයාගේ ඇහැට පෙනෙන ලස්සන හැඩතල (UI) ටික ලියන්නේ. මේක බැලූ බැල්මට HTML වගේ පෙනුණට මේකට කියන්නේ **JSX** කියලා. මේක ඇතුළේ `{pageTitle}` වගේ සඟල වරහන් `{}` පාවිච්චි කරලා අර උඩ ලියපු Variables කෙලින්ම පෙන්නන්න පුළුවන්.

> **සාරාංශය:**
> * **React** කියන්නේ කෝඩ් එක පොඩි කෑලි (Components) වලට කඩලා ලියන ක්‍රමයක්.
> * **Next.js** කියන්නේ මේ React පාවිච්චි කරලා ගොඩක් ලේසියෙන් ෆුල් වෙබ්සයිට් එකක්ම හදාගන්න දෙන Framework එකක් (ඒකෙදි ෆෝල්ඩරේ නම තමයි URL එක වෙන්නේ).
> * හැම Next.js Page එකක්ම **`export default function ...`** කියලා පටන් ගන්න ලොකු ලෙගෝ කෑල්ලක් (Component එකක්).

දැන් ඔයාට Frontend එකේ හැඩය ගැන අවබෝධයක් තියෙනවා. හැබැයි මේ ලෙගෝ කෑලි වලට **"මතක තබාගැනීමේ හැකියාවක්"** දෙන්න ඕනේ. (උදාහරණයක් විදිහට: බට්න් එක කී පාරක් එබුවද කියලා මතක තියාගන්න). අන්න ඒක කරන්නේ කොහොමද කියලා අපි ඊළඟ පාඩමෙන් (React State) ඉගෙන ගනිමු!
