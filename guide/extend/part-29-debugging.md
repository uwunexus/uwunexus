# Part 29: දෝෂ හොයාගන්නේ කොහොමද? (Debugging)

Software Engineering වල ලොකුම රහසක් මම ඔයාට කියන්නම්. **"දක්ෂම Software Engineer ගේ වුණත් කෝඩ් එක පළවෙනි පාරම හරියට වැඩ කරන්නේ නැහැ!"** 

කෝඩ් කරද්දී එක එක ජාතියේ වැරදි (Bugs / Errors) එන එක ගොඩක් සාමාන්‍ය දෙයක්. දක්ෂයෙක් වෙන්නේ ඒ එන වැරැද්ද හොයාගෙන, ඒක නිවැරදි කරගන්න (Debug කරන්න) දන්න කෙනයි. මේ පාඩමෙන් අපි බලමු අපේ Project එකේ වැරැද්දක් ආවොත් ඒක හොයාගන්නේ කොහොමද කියලා.

---

## 1. බ්‍රව්සර් එකේ Console එක සහ Network Tab එක (F12)

මේක තමයි Web Developer කෙනෙක්ගේ හොඳම යාළුවා. ඔයා Google Chrome වගේ බ්‍රව්සර් එකක වෙබ්සයිට් එක බල බල ඉද්දී, කීබෝඩ් එකේ **`F12`** බොත්තම (නැත්නම් Right Click -> Inspect) එබුවම පැත්තකින් අලුත් මෙනු එකක් එනවා (Developer Tools).

### Console Tab එක
Frontend එකේ (React වල) මොනවා හරි වැරැද්දක් වුණොත් (උදා: ඔයා ලියපු Variable එකක නම වැරදියි නම්), ඒක රතු පාටින් මේ Console එකේ පෙන්නනවා.
ඔයාට කෝඩ් එක ඇතුළේ වෙන දේවල් බලාගන්න ඕනේ නම්, කෝඩ් එක ඇතුළේ මෙහෙම ලියන්න පුළුවන්:
```js
console.log("මේක වැඩ කරනවද කියලා බලන්න!");
console.log(myVariable); // Variable එක ඇතුළේ තියෙන දේ බලන්න
```

### Network Tab එක
මේක තමයි ගොඩක්ම වැදගත්! Backend (PHP) එකට යවන දත්ත හරියට යනවද, එතනින් දත්ත එනවද කියලා බලන්නේ මේකෙන්.
* ඔයා Signup බොත්තම එබුවම, මේ Network Tab එකේ පේනවා `signup.php` එකට ගියපු කෝල් එක.
* ඒක උඩ ක්ලික් කළාම ඔයාට පේනවා ඔයා යවපු ඊමේල් එක හරියට ගිහිල්ලද, එහෙන් මොකක්ද ආපු උත්තරේ (Response) කියලා.

## 2. සුලබව එන වැරදි (Common Errors)

**1. `500 Internal Server Error`**
Network Tab එකේ මේක රතු පාටින් දැක්කොත් ඒකෙන් කියන්නේ **"Frontend එකෙන් හරියට දත්ත යැව්වා, ඒත් Backend එකේ (PHP වල) මොකක් හරි කෝඩ් එකක් වැරදිලා එයා කඩන් වැටුණා"** කියන එකයි.
* **විසඳුම:** PHP කෝඩ් එකේ කොමා (,) හරි සෙමිකෝලන් (;) හරි අමතක වෙලාද බලන්න. SQL කමාන්ඩ් එකේ මොනවා හරි වැරදිලාද බලන්න.

**2. `CORS Policy Error`**
Console එකේ "blocked by CORS policy" කියලා දැක්කොත්, ඒකෙන් කියන්නේ Frontend එකට Backend එකෙන් දත්ත ගන්න අවසර නැහැ කියන එකයි.
* **විසඳුම:** `db.php` එකේ තියෙන `Access-Control-Allow-Origin` කියන එක හරියටම තියෙනවද බලන්න.

**3. "සයිට් එක සුදු පාට වෙලා හිරවෙනවා"**
React වලදී මෙහෙම වෙන්නේ ගොඩක් වෙලාවට `map()` එකක් පාවිච්චි කරද්දී, අදාළ දත්තය Array එකක් නොවුණොත් (උදා: ඒක Null වුණොත්). 
* **විසඳුම:** `events.map(...)` වෙනුවට `events?.map(...)` කියලා ප්‍රශ්නාර්ථ ලකුණක් (?) දාන්න. ඒකෙන් කියන්නේ "events තියෙනවා නම් විතරක් map කරන්න" කියන එකයි.

## 3. Backend එක (PHP) Debug කරන හැටි

Frontend එකේ වගේ PHP වලදී අපිට බ්‍රව්සර් Console එකට දේවල් යවන්න බැහැ (මොකද ඒක දුවන්නේ Server එකේ නිසා). 
ඒ නිසා PHP වල මොනවා හරි Variable එකක තියෙන්නේ මොනවද කියලා බලාගන්න අපි පාවිච්චි කරන්නේ `print_r()` හෝ `echo` කියන කමාන්ඩ් එකයි.

```php
<?php
// දත්ත ටික එනවද කියලා බලාගන්න ඕනේ නම්:
$data = json_decode(file_get_contents("php://input"));

// මෙහෙම ලිව්වම, අර Network Tab එකේ Response එකේ මේ විස්තරේ පේනවා
print_r($data); 
?>
```

> **සාරාංශය:**
> * දෝෂ (Bugs) එන එක සාමාන්‍යයි. ඒකට බය වෙන්න එපා.
> * දෝෂයක් ආවම මුලින්ම යන්න ඕනේ බ්‍රව්සර් එකේ **`F12 -> Console / Network`** එකටයි.
> * Frontend එකේ මොනවද තියෙන්නේ බලන්න **`console.log()`** පාවිච්චි කරන්න.
> * Backend එකේ මොනවද තියෙන්නේ බලන්න **`print_r()`** පාවිච්චි කරන්න.

දැන් ඔයා හරියටම දන්නවා ප්‍රශ්නයක් ආවම ඒක හොයාගෙන විසඳගන්නේ කොහොමද කියලා. 

අපේ මාර්ගෝපදේශයේ අවසාන පාඩම වෙන ඊළඟ පාඩමට අපි යමු. ඒකෙන් අපි බලමු **"ඔබ දැන් යන්න ඕනේ කොහාටද? (Mastery)"** කියලා.
