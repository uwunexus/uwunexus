# 🎓 UWU-NEXUS සරල මාර්ගෝපදේශය - (Extend Guide)
## 5 වෙනි කොටස: උත්සව (Events) සහ ටිකට් ගැනීම (Tickets)

විශ්වවිද්‍යාලයේ තියෙන අලුත්ම උත්සව මොනවද කියලා බලාගන්න වගේම, සල්ලි දීලා ටිකට් ගන්නත් අපේ වෙබ් අඩවියෙන් පුළුවන්. මේක ඇතුළේ වැඩ කරන්නේ කොහොමද කියලා අපි සරලව බලමු.

---

### 1. Events (උත්සව) හැදෙන්නේ කොහොමද?

ඕනෑම කෙනෙකුට ඇවිත් වෙබ් අඩවියේ උත්සවයක් දාන්න බෑ. එහෙම දුන්නොත් බොරු උත්සව ගොඩක් පිරෙන්න පුළුවන්. ඒක නිසා අපි මේකට **අනුමත කිරීමේ ක්‍රමයක් (Approval System)** එකක් හදලා තියෙනවා.

**වෙන දේ පියවරෙන් පියවර:**
1. **Club Admin (සංගමයක ප්‍රධානියෙක්)** ඇවිත් අලුත් උත්සවයක විස්තර (නම, දිනය, ෆොටෝ එක) දාලා Save කරනවා.
2. ඒක කෙලින්ම වෙබ් අඩවියේ පේන්නේ නෑ! ඒක Database එකේ සේව් වෙන්නේ **"Pending" (පොරොත්තු වෙමින්)** කියලා.
3. ඊට පස්සේ **Super Admin (ප්‍රධානියා)** Admin Panel එකට ගිහින් ඒක බලනවා.
4. ප්‍රධානියා ඒක **"Approve" (අනුමතයි)** කළොත් විතරක්, ළමයින්ට වෙබ් අඩවියේ Events පිටුවෙන් ඒක පේන්න ගන්නවා.

---

### 2. Free සහ Paid උත්සව අතර වෙනස

අපේ Database එකේ මේකට වගු (Tables) 2ක් තියෙනවා.
- නිකන්ම යන උත්සව සේව් වෙන්නේ **`events`** කියන Table එකේ. (සල්ලි ගෙවන්න නෑ).
- ටිකට් දීලා යන උත්සව සේව් වෙන්නේ **`ticketed_events`** කියන Table එකේ. මේකේ විශේෂත්වය තමයි "ටිකට් ගාණ" සහ "මිල" සේව් කරලා තියෙන එක. (උදාහරණ: ටිකට් 200යි තියෙන්නේ, එකක් රුපියල් 1000යි).

---

### 3. ටිකට් එකක් සල්ලි දීලා ගන්නේ කොහොමද? (Stripe ක්‍රමය)

කෙනෙක් ටිකට් එකක් ගන්නකොට අපි ඒ සල්ලි ගන්නේ **Stripe (ස්ට්‍රයිප්)** කියන ලෝකෙම පිළිගත්ත ආරක්ෂිත ක්‍රමයෙන්. ඒ කියන්නේ ළමයින්ගේ කාඩ්පත් (Card) විස්තර අපි අපේ Database එකේ සේව් කරගන්නේ නෑ.

**ටිකට් එකක් ගන්නකොට වෙන දේ:**

1. **Order එකක් හැදීම**: ළමයෙක් "Proceed to Payment" එබුවම අපේ Backend එක Database එකේ ලියනවා, *"මෙන්න මේ ළමයා ටිකට් එකක් ගන්න හදන්නේ, හැබැයි තාම සල්ලි දුන්නේ නෑ (Pending)"* කියලා. එතකොට අපේ Backend එකෙන් ඒ ළමයාට Order ID එකක් දෙනවා (උදාහරණ: TKT-12345).
2. **Stripe එකට යැවීම**: ඊට පස්සේ ළමයාව කෙලින්ම අපේ වෙබ් අඩවියෙන් Stripe එකේ ආරක්ෂිත පිටුවට (Secure Page) යවනවා කාඩ් එකේ විස්තර ගහන්න. 
3. **සල්ලි කැපීම**: ළමයා එතන කාඩ් විස්තර ගහලා සල්ලි ගෙව්වම, Stripe එකෙන් අපේ Backend එකට රහස් කෝල් එකක් දෙනවා (මේකට කියන්නේ Webhook කියලා), *"ඔන්න අර TKT-12345 කියන කෙනා සල්ලි දුන්නා!"* කියලා.
4. **Database එක Update වීම**: 
   - එතකොට අපේ Backend එක ආපහු Database එකට ගිහින් අර ළමයාගේ Order එක "Pending" ඉඳන් **"Success" (සාර්ථකයි)** කියලා වෙනස් කරනවා.
   - ඒ වගේම ටිකට් ගාණ අඩු කරනවා. (කලින් ටිකට් 200ක් තිබ්බා නම්, දැන් 199යි කියලා Database එකේ අලුත් කරනවා).
5. **Sold Out (ඉවරයි)**: ටිකට් ගාණ 0 වුණාම, Frontend එකෙන් "Buy Ticket" බොත්තම අක්‍රිය කරලා "Sold Out" කියලා පෙන්නනවා.

---

### 4. ඔයාට වෙනස්කමක් කරන්න ඕනේ නම්?

හිතන්න ඔයාට කවුරුහරි කිව්වොත්: 
*"ළමයෙක් සල්ලි දීලත් එයාගේ ටිකට් එක 'Pending' කියලා වැටිලා තියෙනවා (Stripe එකෙන් කෝල් එක ආවේ නැති වුණොත් එහෙම වෙන්න පුළුවන්), ඒක හදන්න ඕනේ!"*

1. Database එකේ **`ticket_purchases`** කියන Table එකට යන්න.
2. අදාළ ළමයාගේ නම හරි Order ID එක හරි තියෙන පේළිය හොයාගන්න.
3. ඒකේ `status` කියන තීරුවේ තියෙන "pending" කියන වචනේ මකලා "success" කියලා Type කරලා save කරන්න.
4. ඊට පස්සේ **`ticketed_events`** කියන Table එකට ගිහින් ඒ උත්සවයේ ටිකට් ගාණෙන් එකක් අඩු කරලා save කරන්න. 
(ඔයා දැන් Programming නොදැනම ලොකු දෝෂයක් (Bug) එකක් හැදුවා!)
