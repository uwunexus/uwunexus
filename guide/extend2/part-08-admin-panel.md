# 🎓 UWU-NEXUS සරල මාර්ගෝපදේශය - (Extend Guide)
## 8 වෙනි කොටස: Admin Panel එක (වෙබ් අඩවියේ පාලක මැදිරිය)

වෙබ් අඩවියක් හැදුවට පස්සේ ඒක පාලනය කරන්න, වැරදි දේවල් අයින් කරන්න, අලුත් දේවල් අනුමත කරන්න කෙනෙක් ඉන්න ඕනෙනේ. අන්න ඒ වැඩේ කරන්නේ **Admin Panel** එකෙන්. මේක හරියට වෙබ් අඩවියේ "පාලක මැදිරිය" වගේ.

---

### 1. කවුද මේ Admins ලා? (තනතුරු)

අපේ වෙබ් අඩවියේ ප්‍රධාන තනතුරු (Roles) කිහිපයක් තියෙනවා:
1. **Super Admin (ප්‍රධානියා)**: මුළු වෙබ් අඩවියම පාලනය කරන්න පුළුවන් කෙනා. ඕනෑම දෙයක් මකන්න, වෙනස් කරන්න පුළුවන්.
2. **Club Admin (සංගම් ප්‍රධානියා)**: මෙයාලට පුළුවන් විශ්වවිද්‍යාලයේ උත්සව (Events) දාන්න විතරයි.
3. **Student (සාමාන්‍ය ළමයා)**: මෙයාලට අනිත් අය දාන දේවල් බලන්න, බඩු විකුණන්න දාන්න පුළුවන් වුණාට, අනුමත කිරීම් (Approvals) කරන්න බෑ.

---

### 2. Admin Panel එකෙන් කරන්න පුළුවන් දේවල්

Super Admin කෙනෙක් `/admin` කියන පිටුවට ගියාම එයාට Tabs (කොටස්) 6ක් පේනවා.

1. **Users (පරිශීලකයින්)**: වෙබ් අඩවියේ ඉන්න හැම ළමයෙක්ගෙම විස්තර බලාගන්න, එයාලව Admin ලා කරන්න, නැත්තම් අකවුන්ට් එක මකලා දාන්න (Delete) පුළුවන්.
2. **Events (උත්සව)**: Club Admin ලා දාන උත්සව බලලා "Approve" (අනුමතයි) හරි "Reject" (ප්‍රතික්ෂේපයි) හරි කරන්න පුළුවන්.
3. **Tickets (ටිකට්)**: සල්ලි දීලා ටිකට් ගත්ත අය කවුද කියලා බලාගන්න පුළුවන්.
4. **Marketplace (වෙළඳපොළ)**: ළමයි විකුණන්න දාන බඩු අනුමත කරන්න පුළුවන්.
5. **Lost & Found (නැතිවූ දේවල්)**: පරණ වුණ, විසඳපු ප්‍රශ්න මකලා දාන්න පුළුවන්.
6. **Info Hub (තොරතුරු පියස)**: විශ්වවිද්‍යාලයේ හදිසි දුරකථන අංක, නීතිරීති වගේ දේවල් වෙබ් අඩවියට එකතු කරන්න පුළුවන්.

---

### 3. සාමාන්‍ය ළමයෙක් Admin Panel එකට ගියොත්? (Security)

සාමාන්‍ය ළමයෙක් තනියම බ්‍රව්සර් එකේ `/admin` කියලා ගහලා ගියොත් මොකද වෙන්නේ? එයාට ඇතුළට යන්න පුළුවන්ද? **බැහැ!**

අපි මේකට **ආරක්ෂක දැල් (Security Layers) 2ක්** දාලා තියෙනවා:

1. **පළවෙනි දැල (Frontend Security)**:
   - ළමයෙක් `/admin` පිටුවට යන්න හැදුවම, අපේ Frontend එක (Next.js) ළමයාගේ ID කාඩ් එක (Cookie එක) බලනවා.
   - එතන 'Student' කියලා තිබ්බොත්, එයාට ඒ පිටුව පෙන්නන්නේ නැතුව ආපහු Home Page එකට විසි කරනවා! (Redirect කරනවා).

2. **දෙවෙනි දැල (Backend Security)**:
   - කවුරුහරි හැකර් කෙනෙක් පළවෙනි දැල කඩාගෙන Database එකෙන් ළමයෙක්ව මකන්න (Delete) ට්‍රයි කළා කියමු.
   - අපේ Backend එක (PHP) ඒක කරන්න කලින් ආයෙත් Database එකට ගිහින් අහනවා *"මේ මකන්න කියන කෙනා ඇත්තටම Super Admin කෙනෙක්ද?"* කියලා.
   - එතන 'Student' කියලා තිබ්බොත්, Backend එක ඒ රික්වෙස්ට් එක ප්‍රතික්ෂේප කරලා "Forbidden (තහනම්)" කියලා එවනවා.

මේ දැල් දෙක නිසා කිසිම කෙනෙක්ට හොරෙන් Admin වැඩ කරන්න බෑ!

---

### 4. Info Hub එක කියන්නේ මොකක්ද?

ඔයාට කවදාහරි විශ්වවිද්‍යාලයේ මෙඩිකල් එකක් දාගන්නේ කොහොමද, නැත්තම් Security එකේ නම්බර් එක මොකක්ද කියලා දැනගන්න ඕනෙ වෙලා තියෙනවද? 

අන්න ඒ වගේ **වැදගත් තොරතුරු (Procedures, Hotlines, Contacts)** ළමයින්ට ලේසියෙන් බලාගන්න හදපු තැන තමයි Info Hub එක. Admin Panel එකට ගිහින් Admin කෙනෙක්ට පුළුවන් අලුත් ෆෝම් එකක ලින්ක් එකක් හරි, අලුත් නම්බර් එකක් හරි ලේසියෙන්ම වෙබ් අඩවියට එකතු කරන්න.

---

### 5. ඔයාට වෙනස්කමක් කරන්න ඕනේ නම්?

හිතන්න ඔයාට කවුරුහරි කිව්වොත්:
*"අපිට Admin Panel එකේ Users ලා පේන තැන ළමයාගේ නමට පස්සේ එයාගේ වයසත් පේන්න හදන්න ඕනේ!"*

1. මේක Frontend එකේ පෙනුම වෙනස් කරන වැඩක්.
2. `app/admin` ෆෝල්ඩරයට ගිහින් `page.tsx` ෆයිල් එක අරින්න.
3. ඒකේ Users ලාව පෙන්නන තැන තියෙන කෝඩ් එක හොයාගන්න (Table එකක් වගේ තියෙන්නේ).
4. ඒකේ අලුත් තීරුවක් (Column එකක්) එකතු කරලා `user.age` කියලා ලියන්න.
(මෙතනදී Database එකේ Users වගුවෙත් වයස සේව් කරන්න තැනක් තියෙන්න ඕනේ කියලා ඔයාට දැන් තේරෙනවා ඇති!)

---
**දැන් ඔයාට මුළු වෙබ් අඩවියම පාලනය කරන Admin Panel එක සහ ඒකේ ආරක්ෂාව ගැන හොඳ අදහසක් තියෙනවා.**

ඊළඟ (9 වෙනි) කොටසෙන් අපි කතා කරමු ඔයාගේ පරිගණකයේ තියෙන මේ වෙබ් අඩවිය, **මුළු ලෝකෙටම පේන්න අන්තර්ජාලයට දාන්නේ (Deployment) කොහොමද** කියලා!
