# 🎓 UWU-NEXUS සරල මාර්ගෝපදේශය - (Extend Guide)
## 15 වෙනි කොටස: එක එක අයට වෙබ් අඩවිය වෙනස් විදිහට පේන්නේ කොහොමද? (Roles & Views)

ඔයා දැකලා ඇති සාමාන්‍ය ළමයෙක් ලොග් වුණාම එයාට පේන දේවල් නෙවෙයි, Admin කෙනෙක් ලොග් වුණාම පේන්නේ. ඒ වගේම Admin Panel එක ඇතුළට ගියාමත් Club Admin ට පේන දේවල් නෙවෙයි Super Admin ට පේන්නේ.

මේක හරියට **තනතුරු (Roles)** අනුව දොරවල් ඇරෙනවා වගේ වැඩක්. මේක කොහොමද කරලා තියෙන්නේ කියලා අපි සරලව බලමු.

---

### 1. තනතුර (Role) මතක තියාගන්නේ කොහෙද?

අපි කලින් කතා කළා මතකද "Cookies (ID කාඩ් එක)" ගැන? කෙනෙක් වෙබ් අඩවියට ලොග් වෙනකොට, Backend එකෙන් එයාට දෙන Cookie එක ඇතුළේ එයාගේ තනතුර ලියලා තියෙනවා.

- සාමාන්‍ය ළමයෙක්ට: `"role": "student"`
- සංගමයක ප්‍රධානියෙක්ට: `"role": "club_admin"`
- ප්‍රධාන ඇඩ්මින්ට: `"role": "superadmin"`

Frontend එක (Next.js) මේ ID කාඩ් එක කියවලා තේරුම් ගන්නවා දැන් ලොග් වෙලා ඉන්නේ කවුද කියලා.

---

### 2. සාමාන්‍ය ළමයා vs Admin (පේන දේ වෙනස් වෙන හැටි)

හිතන්න වෙබ් අඩවියේ උඩින්ම තියෙන මෙනුව (Navigation Bar) ගැන.
සාමාන්‍ය ළමයෙක්ට ඒකේ පේන්නේ `Home`, `Events`, `Marketplace` වගේ දේවල් විතරයි. හැබැයි Admin කෙනෙක්ට එතන **`Admin Panel`** කියලා අලුත් බොත්තමක් පේනවා.

මේක Frontend එකේ කෝඩ් කරලා තියෙන්නේ හරියට **"ඔව් ද? නැද්ද?" (If-Else)** කියන තර්කය පාවිච්චි කරලයි. 

*කෝඩ් එක ඇතුළේ තියෙන ලොජික් එක සරලව කිව්වොත් මෙහෙමයි:*
```text
(ID කාඩ් එක බලන්න...)
තනතුර 'superadmin' හෝ 'club_admin' ද?
   ඔව් නම් -> "Admin Panel" බොත්තම පෙන්නන්න!
   නැත්නම් -> මුකුත් පෙන්නන්න එපා.
```
මෙන්න මේ පොඩි ලොජික් එක නිසා තමයි ළමයින්ට අර බොත්තම හැංගිලා ඇඩ්මින්ලට විතරක් ඒක පේන්නේ.

---

### 3. Club Admin සහ Super Admin අතර වෙනස

Admin Panel එක ඇතුළට ගියාමත් මේ දෙගොල්ලන්ට පේන්නේ කොටස් (Tabs) දෙවිදිහකට.

- **Super Admin ට**: Users (ළමයි අයින් කරන්න), Events (උත්සව අනුමත කරන්න), Marketplace (බඩු අනුමත කරන්න), Lost & Found... මේ ඔක්කොම ටැබ්ස් (Tabs) 6ම පේනවා.
- **Club Admin ට**: එයාලට ළමයි අයින් කරන්න අයිතියක් නෑනේ. ඒක නිසා එයාලට පේන්නේ **Events** ටැබ් එක විතරයි!

මේකත් Frontend එකේ හදලා තියෙන්නේ අර වගේම තර්කයකින් (If-Else):
```text
(ID කාඩ් එක බලන්න...)
තනතුර 'superadmin' ද?
   ඔව් නම් -> Users ටැබ් එක පෙන්නන්න.
තනතුර 'club_admin' ද?
   ඔව් නම් -> Events ටැබ් එක විතරක් පෙන්නන්න.
```
ඒක නිසා Club Admin කෙනෙක් ලොග් වුණ ගමන් එයාට අනිත් දේවල් ඔක්කොම හැංගෙනවා. එයාට පුළුවන් එයාගේ උත්සව (Events) ටික විතරක් බලාගන්න.

---

### 4. Backend එකෙන් දෙන ආරක්ෂාව!

හිතන්න Club Admin කෙනෙක් ටිකක් කපටියි කියලා. එයා කෝඩින් දන්නවා නම්, එයාට පුළුවන් කෝඩ් එක පොඩ්ඩක් වෙනස් කරලා අර හැංගිලා තියෙන "Users" ටැබ් එක බලෙන් මතු කරගන්න. එතකොට එයාට පුළුවන්ද වෙන ළමයෙක්ව මකලා දාන්න (Delete)?

**බැහැ!** මොකද අපි මේකට Backend එකෙත් (කුස්සියේත්) ආරක්ෂාවක් දාලා තියෙනවා.

Frontend එකෙන් කෝල් එකක් ආවොත් *"නිලේෂ් කියන ළමයාව මකන්න"* කියලා, 
Backend එක ඒ වැඩේ කරන්න කලින්, එවන කෙනාගේ ID කාඩ් එක ආයේ සැරයක් චෙක් කරනවා.
එතන `"role": "club_admin"` කියලා තිබ්බොත්, Backend එක කියනවා:
*"සමාවෙන්න! ඔයාට ළමයි මකන්න අයිතියක් නෑ. මේක කරන්න පුළුවන් Super Admin ට විතරයි!"* කියලා කෝල් එක විසන්ධි කරලා දානවා (403 Forbidden Error).

### 5. ඔයාට වෙනස්කමක් කරන්න ඕනේ නම්?

හිතන්න ඔයාට කවුරුහරි කිව්වොත්:
*"දැන් ඉඳන් Club Admin ලටත් Marketplace එක (බඩු අනුමත කරන තැන) පේන්න හදන්න ඕනේ!"*

1. යන්න ඕනේ Frontend එකේ `app/admin/page.tsx` ෆයිල් එකට.
2. එතන තියෙනවා Marketplace ටැබ් එක පෙන්නන කෝඩ් එකක්.
3. ඒකේ ලියලා ඇති `role === 'superadmin'` (සුපර් ඇඩ්මින් නම් විතරක් පෙන්නන්න) කියලා.
4. ඒක වෙනස් කරලා `role === 'superadmin' || role === 'club_admin'` කියලා ලියලා සේව් කරන්න. 
(ඒකෙන් කියන්නේ සුපර් ඇඩ්මින් හරි ක්ලබ් ඇඩ්මින් හරි දෙන්නගෙන් කවුරු ආවත් පෙන්නන්න කියලා).
5. ඊට පස්සේ Backend එකේ Marketplace අනුමත කරන PHP ෆයිල් එකටත් ගිහින් අර වගේම Club Admin ටත් අවසර දෙන්න ඕනේ.
