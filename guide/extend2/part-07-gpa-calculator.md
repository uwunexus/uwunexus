# 🎓 UWU-NEXUS සරල මාර්ගෝපදේශය - (Extend Guide)
## 7 වෙනි කොටස: GPA Calculator එක (ප්‍රතිඵල ගණනය කිරීම)

ළමයින්ට තමන්ගේ විෂයයන් (Subjects) වලට ගත්ත සාමාර්ථ (Grades) දාලා තමන්ගේ මුළු ප්‍රතිඵලය (GPA එක) බලාගන්න පුළුවන් තැනක් තමයි මේක. මේක ටිකක් සංකීර්ණ වුණාට අපි සරලව තේරුම් ගමු.

---

### 1. වෙබ් අඩවිය ඔයාගේ විෂයයන් දන්නේ කොහොමද?

ඔයා GPA පිටුවට ගියාම ඔයාගේ Degree එකට (උදාහරණ: IIT) අදාළ අවුරුදු 4ටම තියෙන විෂයයන් ටික ඔටෝම එතන පෙන්නනවා. ඒක වෙන්නේ මෙහෙමයි:

1. ඔයා ලොග් වෙනකොට ඔයාගේ Email එක (`iit23068@std.uwu.ac.lk`) Backend එකෙන් කියවනවා.
2. ඒකේ මුල තියෙන අකුරු 3න් (iit) Backend එක තේරුම් ගන්නවා ඔයාගේ Degree එක IIT කියලා.
3. ඊට පස්සේ Database එකට ගිහින් IIT degree එකට අදාළ ඔක්කොම විෂයයන් ටික අරගෙන ඇවිත් තිරයේ පෙන්නනවා.

### 2. මේකට සම්බන්ධ Database වගු (Tables) 4

මේ වැඩේ කරන්න Database එකේ වගු 4ක් එකට එකතු වෙලා වැඩ කරනවා. මේක හරියට ෆයිල් කබඩ් එකක් වගේ හිතන්න.

1. **`degrees` වගුව**: මේකේ තියෙන්නේ විශ්වවිද්‍යාලයේ තියෙන Degree වල නම්. (උදා: IIT, CST).
2. **`modules` වගුව**: මේකේ තියෙන්නේ උගන්නන හැම විෂයයකම නම් සහ ඒකට තියෙන Credits (බර තැබීම).
3. **`curriculum_groups` වගුව**: මේකෙන් තමයි කියන්නේ "මෙන්න මේ විෂය යන්නේ 1 වෙනි අවුරුද්දේ 1 වෙනි සෙමෙස්ටර් එකට" කියලා.
4. **`user_grades` වගුව**: මේකේ තමයි ළමයි ගන්න Grades (A, B, C) සේව් කරලා තියාගන්නේ.

### 3. GPA එක ගණනය කරන්නේ කොහොමද? (The Math)

GPA එක කියන්නේ 0.00 ඉඳන් 4.00 ට තියෙන අගයක්. ඔයා ගන්න අකුරු වලට (Grades) වටිනාකමක් තියෙනවා:
- A+ / A = 4.00
- B = 3.00
- C = 2.00
- E (Fail) = 0.00

**Credits (ක්‍රෙඩිට්ස්) කියන්නේ මොකක්ද?**
හැම විෂයයක්ම එක වගේ නෑ. අමාරු/ලොකු විෂයයන් වලට Credits වැඩියි (උදා: 3). පොඩි විෂයයන් වලට Credits අඩුයි (උදා: 2).
ඔයා Credits 3ක විෂයයකට 'A' එකක් ගත්තොත්, ඒක ඔයාගේ GPA එකට ගොඩක් හොඳට බලපානවා. 

Backend එකෙන් මේ ඔක්කොම එකතු කරලා, බෙදලා තමයි අන්තිම උත්තරේ (උදා: 3.25) ඔයාට පෙන්නන්නේ. ඒ අගය අනුව ඔයාට First Class ද, Second Class ද කියලත් පෙන්නනවා!

### 4. විෂයයක් Fail වුණොත් (Repeat කළොත්) මොකද වෙන්නේ?

හිතන්න ළමයෙක් විෂයයකට 'E' (Fail) එකක් අරගෙන, ඊළඟ අවුරුද්දේ ඒක ආයේ කරලා 'C' එකක් ගන්නවා කියලා. 

අපේ **`user_grades`** වගුවේ විශේෂ තීරුවක් (Column එකක්) තියෙනවා `is_best_grade` (හොඳම ප්‍රතිඵලය) කියලා.
1. මුලින් ගත්ත 'E' එක සේව් වෙනවා. 
2. ඊළඟ අවුරුද්දේ 'C' එක ගත්තම ඒක අලුත් පේළියක සේව් වෙනවා.
3. ඊට පස්සේ Backend එක අර පරණ 'E' එක අතහැරලා, අලුත් 'C' එක ඉස්සරහින් විතරක් `is_best_grade = 1` කියලා ලියනවා.
4. GPA එක හදද්දී ගණන් ගන්නේ අර හොඳම ප්‍රතිඵලය විතරයි!

### 5. ඔයාට වෙනස්කමක් කරන්න ඕනේ නම්?

හිතන්න ඔයාට කවුරුහරි කිව්වොත්:
*"විශ්වවිද්‍යාලයෙන් අලුත් Degree එකක් පටන් අරන් (උදා: Data Science - DST). ඒක Database එකට දාන්න!"*

1. Database එකේ **`degrees`** කියන Table එක අරින්න.
2. ඒකේ අලුත් පේළියක් (Row එකක්) එකතු කරන්න.
3. `degree_code` එකට "DST" කියලත්, `degree_name` එකට "Data Science" කියලත් ලියලා Save කරන්න.
4. එච්චරයි! දැන් ඒ Degree එකේ ළමයින්ටත් වෙබ් අඩවියට ලොග් වෙලා GPA එක හදාගන්න පුළුවන්.
