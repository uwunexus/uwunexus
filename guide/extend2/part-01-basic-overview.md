# 🎓 UWU-NEXUS සරල මාර්ගෝපදේශය - (Extend Guide)
## 1 වෙනි කොටස: වෙබ් අඩවියක් වැඩ කරන්නේ කොහොමද සහ මේ Project එක මොකක්ද?

කෝඩින් (Coding) ගැන කිසිම දෙයක් නොදන්න ඔයාට මේ මුළු project එකම තේරුම් ගන්න පුළුවන් වෙන විදිහට තමයි මේක ලියලා තියෙන්නේ. මේක හරියට කතාවක් වගේ කියවන්න!

---

### 1. මේ Project එක මොකක්ද?
මේක Uva Wellassa University (UWU) එකේ ළමයින්ට පාවිච්චි කරන්න හදපු **වෙබ් අඩවියක්**. 
මේකෙන් ළමයින්ට පුළුවන්:
- විශ්වවිද්‍යාලයේ තියෙන Events (උත්සව) ගැන දැනගන්න සහ ටිකට් ගන්න.
- පරණ පොත්, බඩු විකුණන්න සහ ගන්න (Marketplace).
- නැතිවුණ බඩු ගැන දාන්න (Lost & Found).
- තමන්ගේ GPA එක (Result) ගණනය කරගන්න.

### 2. වෙබ් අඩවියක් හැදිලා තියෙන ප්‍රධාන කොටස් 3
ඕනෑම ලොකු වෙබ් අඩවියක් (Facebook වගේ) කොටස් 3කින් තමයි හැදිලා තියෙන්නේ. අපේ එකෙත් එහෙමයි.

1. **Frontend (ඉස්සරහා පෙනෙන කොටස)**
   - මේක හරියට **රෙස්ටුරන්ට් එකක කෑම කන තැන** වගේ. 
   - අපිට පේන ලස්සන පාට, බොත්තම්, පින්තූර ඔක්කොම තියෙන්නේ මෙතන.
   - අපේ project එකේ මේක හදලා තියෙන්නේ **Next.js** කියන තාක්ෂණයෙන්.

2. **Backend (පිටිපස්සේ තියෙන මොළය)**
   - මේක හරියට **රෙස්ටුරන්ට් එකක කුස්සිය** වගේ.
   - ඉස්සරහින් කෑම order එකක් දුන්නම, ඒක හදලා දෙන්නේ කුස්සියෙන්. ඒ වගේම අපි බොත්තමක් එබුවම ඒකට අදාළ වැඩේ කරන්නේ Backend එකෙන්.
   - අපේ project එකේ මේක හදලා තියෙන්නේ **PHP** කියන භාෂාවෙන්.

3. **Database (දත්ත ගබඩාව / මතක පොත)**
   - මේක හරියට **රෙස්ටුරන්ට් එකේ බඩු ගබඩාව** වගේ. 
   - මිනිස්සුන්ගේ නම්, පාස්වර්ඩ්, දාපු පින්තූරවල නම් මේ ඔක්කොම ලියලා තියාගන්න පොත තමයි Database එක.
   - අපේ project එකේ මේක හදලා තියෙන්නේ **MySQL** කියන තාක්ෂණයෙන්.

### 3. Folder එක ඇතුළේ තියෙන්නේ මොනවාද?
ඔයාගේ computer එකේ මේ project ෆෝල්ඩරය (uwunexus) ඇරියම ෆෝල්ඩර් ගොඩක් පේනවා. බයවෙන්න එපා, අපිට වැදගත් වෙන්නේ ප්‍රධාන ෆෝල්ඩර් දෙකයි:

📁 **`app` ෆෝල්ඩරය** (Frontend එක)
- වෙබ් අඩවියේ පේන හැම පිටුවක්ම (pages) තියෙන්නේ මෙතන. 
- උදාහරණයක් විදිහට කවුරුහරි කිව්වොත් *"Login වෙන පිටුවේ පාට වෙනස් කරන්න"* කියලා, ඔයා යන්න ඕනේ මේ `app` ෆෝල්ඩරය ඇතුළේ තියෙන `login` ෆෝල්ඩරයට. ඒක ඇතුළේ තියෙනවා `page.tsx` කියලා ෆයිල් එකක්. අන්න ඒකෙන් තමයි ඒ පිටුව හැදිලා තියෙන්නේ.

📁 **`backend` ෆෝල්ඩරය** (Backend එක)
- දත්ත ගබඩාවත් එක්ක කතා කරන ෆයිල්ස් ඔක්කොම තියෙන්නේ මෙතන.
- මේවා සේරම `.php` ෆයිල්ස්. (උදාහරණ: `login.php`, `get_events.php`).

---

### 4. මේ තුන එකට කතා කරන්නේ කොහොමද? 
මේක තේරුම් ගන්න අපි උදාහරණයක් ගමු. **කෙනෙක් ලොග් (Login) වෙනකොට වෙන දේ:**

1. **(Frontend එකේ)**: කෙනෙක් එයාගේ Email එකයි Password එකයි ගහලා "Login" බොත්තම ඔබනවා. (මේක වෙන්නේ `app/login/page.tsx` ෆයිල් එකේ)
2. **(යැවීම)**: Frontend එකෙන් මේ Email එකයි Password එකයි පාර්සලයක් වගේ කරලා Backend එකට යවනවා. 
3. **(Backend එකේ)**: Backend එකේ තියෙන `backend/login.php` කියන ෆයිල් එක ඒ පාර්සලය අරගෙන Database එකෙන් අහනවා *"මේ වගේ කෙනෙක් අපේ පොතේ ඉන්නවද?"* කියලා.
4. **(Database එක)**: Database එක පොත පෙරලලා බලලා කියනවා *"ඔව්, ඉන්නවා, පාස්වර්ඩ් එකත් හරි!"* කියලා.
5. **(පිළිතුර)**: Backend එක ආපහු Frontend එකට කියනවා *"ඔක්කොම හරි, ඇතුළට ගන්න"* කියලා. 
6. **(Frontend එකේ)**: ඊට පස්සේ Frontend එකෙන් එයාව සාර්ථකව ඇතුළට අරගෙන Home පිටුව පෙන්නනවා.

### 5. ඔයාට වෙනස්කම් කරන්න ඕනෙ නම් කොහොමද කරන්නේ?

ඔයාට කවුරුහරි කිව්වොත්: 
*"අපිට Events පිටුවේ ලියලා තියෙන වචනයක් වෙනස් කරන්න ඕනේ"*
1. මේක පේන දෙයක් වෙනස් කරන එකක්. ඒ කියන්නේ යන්න ඕනේ Frontend එකට (`app` ෆෝල්ඩරයට).
2. ඒක ඇතුළේ තියෙන `events` ෆෝල්ඩරයට ගිහින් `page.tsx` කියන ෆයිල් එක අරින්න.
3. ඒකේ තියෙන ඉංග්‍රීසි වචන ටික කියවන් ගියාම ඔයාට අදාළ වචනේ හොයාගෙන මකලා අලුත් වචනේ ලියන්න පුළුවන්!

ඔයාට කවුරුහරි කිව්වොත්: 
*"Login වෙනකොට වැරදි පාස්වර්ඩ් එකක් ගැහුවොත් එන Error Message එක වෙනස් කරන්න ඕනේ"*
1. මේක දත්ත පරීක්ෂා කරන තැනක්. ඒ කියන්නේ යන්න ඕනේ Backend එකට (`backend` ෆෝල්ඩරයට).
2. ඒකේ තියෙන `login.php` ෆයිල් එක අරින්න.
3. ඒක ඇතුළේ ලියලා තියෙනවා "Invalid password" වගේ මැසේජ් එකක්. ඒක හොයාගෙන වෙනස් කරන්න.

ඊළඟ (2 වෙනි) කොටසෙන් අපි කතා කරමු **Database (දත්ත ගබඩාව)** කියන පොත ඇතුළේ දේවල් ලියලා තියෙන්නේ කොහොමද සහ ඒක කියවන්නේ කොහොමද කියලා.
