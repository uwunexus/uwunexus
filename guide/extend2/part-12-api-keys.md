# 🎓 UWU-NEXUS සරල මාර්ගෝපදේශය - (Extend Guide)
## 12 වෙනි කොටස: API Keys කියන්නේ මොනවද? (රහස් යතුරු)

අපේ වෙබ් අඩවියේ සල්ලි ගෙවන්න අපි පාවිච්චි කළේ **Stripe** එක, ෆොටෝස් සේව් කරන්න පාවිච්චි කළේ **Cloudinary** එක. මේවා අපේ වෙබ් අඩවියෙන් පිට තියෙන, වෙනමම කම්පැණි වල සර්වර්ස්. 

අපේ වෙබ් අඩවිය මේ වෙන කම්පැණි වල සර්වර්ස් එක්ක කතා කරන්නේ කොහොමද? අන්න ඒකට තමයි **API Keys** පාවිච්චි කරන්නේ.

---

### 1. API Key එකක් කියන්නේ හරියට "VIP පාස් එකක්" වගේ!

හිතන්න ඔයාට ලොකු හෝටලයක තියෙන VIP කාමරයකට යන්න ඕනේ කියලා. එතන ඉන්න ගාඩ් ඔයාට ඇතුළට යන්න දෙන්නේ නෑ ඔයාගෙන් VIP පාස් එකක් ඉල්ලනවා.

අපේ වෙබ් අඩවියත් Cloudinary එකට කෝල් කරලා කිව්වොත් *"මෙන්න ෆොටෝ එකක් ආවා, මේක ඔයාගේ සර්වර් එකේ සේව් කරගන්න"* කියලා, Cloudinary එක අහනවා *"ඔයා කවුද? මට ඔයාව විශ්වාස නෑ, ඔයාගේ VIP පාස් එක පෙන්නන්න!"* කියලා.

අන්න ඒ VIP පාස් එක තමයි **API Key** එක කියන්නේ. ඒක ඉලක්කම් සහ අකුරු ගොඩක් තියෙන දිග කෝඩ් එකක්. (උදාහරණ: `sk_test_51AbcDefGhi12345...`)

---

### 2. අපි කොහොමද මේ යතුරු පාවිච්චි කරන්නේ?

අපි Stripe (සල්ලි ගෙවන තැන) අකවුන්ට් එකක් හැදුවම, එයාලා අපිට දෙනවා එයාලගේ සර්වර් එකට ඇතුල් වෙන්න පුළුවන් API Key එකක්.
ඒකෙ යතුරු දෙකක් තියෙනවා:
1. **Public Key (පොදු යතුර)**: මේක ඕනෑම කෙනෙක්ට දැක්කට කමක් නෑ. මේක අපි පාවිච්චි කරන්නේ Frontend එකේ (පේන පැත්තේ). ළමයින්ගේ කාඩ් විස්තර Stripe එකට යවන්නේ මේ යතුරෙන්.
2. **Secret Key (රහස් යතුර)**: මේක ගොඩක් භයානකයි. මේක හැකර් කෙනෙක්ට ගියොත් එයාට අපේ සල්ලි ඔක්කොම ගන්න පුළුවන්. ඒක නිසා මේක අපි කවදාවත් Frontend එකේ දාන්නේ නෑ. මේක තියාගන්නේ Backend එකේ (පිටිපස්සේ) විතරයි.

---

### 3. API Keys හංගන්නේ කොහොමද? (Environment Variables)

අපි කලින් කොටසක කතා කළා මතකද "රහස් පෙට්ටි" ගැන? ඒවට කිව්වේ Environment Variables (`.env` ෆයිල්) කියලා.

අපි කවදාවත් අපේ කෝඩ් එක ඇතුළේ:
`$stripe_password = "sk_test_123456";` කියලා ලියන්නේ නෑ! 

ඒ වෙනුවට අපි කරන්නේ මේ රහස් යතුර අර Vercel එකේ හරි අපේ VPS සර්වර් එකේ හරි තියෙන "රහස් පෙට්ටියට" දානවා. ඊට පස්සේ කෝඩ් එකේ ලියනවා:
`$stripe_password = රහස්_පෙට්ටියෙන්_Stripe_යතුර_ගන්න();` කියලා.

එතකොට කවුරුහරි අපේ කෝඩ් එක බැලුවත්, එයාට පේන්නේ නෑ ඇත්තටම පාස්වර්ඩ් එක මොකක්ද කියලා. ඒක දන්නේ සර්වර් එක විතරයි!

---

### 4. ඔයාට වෙනස්කමක් කරන්න ඕනේ නම්?

හිතන්න ඔයාට කවුරුහරි කිව්වොත්:
*"අපි Stripe එකේ අලුත් අකවුන්ට් එකක් හැදුවා. පරණ එක අයින් කරලා මේ අලුත් API Key එක දාන්න!"*

1. මේක කෝඩ් එක ඇතුළේ වෙනස් කරන දෙයක් නෙවෙයි.
2. ඔයාට යන්න වෙන්නේ සර්වර් එකට (Vercel එකට හෝ VPS එකට).
3. එතන තියෙනවා **Environment Variables** (නැත්තම් `config-prod.php` වගේ ෆයිල් එකක්).
4. එතන තියෙන පරණ දිග කෝඩ් එක මකලා, අලුත් කෝඩ් එක දාලා සේව් කරන්න. එතකොට වෙබ් අඩවිය අලුත් යතුරෙන් වැඩ කරන්න පටන් ගන්නවා!

---
**දැන් ඔයාට Cookies වලින් වෙබ් අඩවියක් ඔයාව මතක තියාගන්න විදිහත්, API Keys වලින් වෙනත් වෙබ් අඩවි එක්ක ආරක්ෂිතව කතා කරන විදිහත් ගැන සම්පූර්ණ අදහසක් තියෙනවා!**
