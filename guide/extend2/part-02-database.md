# 🎓 UWU-NEXUS සරල මාර්ගෝපදේශය - (Extend Guide)
## 2 වෙනි කොටස: Database එක (දත්ත ගබඩාව) තේරුම් ගනිමු!

පළවෙනි කොටසේදී අපි කිව්වනේ Database එක කියන්නේ "මතක තියාගන්න පොත" කියලා. අද අපි බලමු මේ පොත ඇතුළේ දේවල් ලියලා තියෙන්නේ කොහොමද කියලා.

---

### 1. Database එකක් කියන්නේ හරියට Excel Sheet එකක් වගේ!

ඔයා කවදාහරි Microsoft Excel පාවිච්චි කරලා තියෙනවද? Database එකක් කියන්නෙත් හරියට ඒ වගේම එකක් තමයි.
අපේ project එකේ පාවිච්චි කරන්නේ **MySQL** කියන Database වර්ගය.

මේකෙදි අපි දත්ත තියාගන්නේ **Tables (වගු)** විදිහට.
හිතන්න අපිට ළමයි ගැන විස්තර තියාගන්න ඕනේ නම්, අපි "Users" කියලා වගුවක් (Table එකක්) හදනවා.

ඒ වගුවේ තියෙනවා:
- **Columns (තීරු)**: නම, වයස, Email එක, Password එක වගේ මාතෘකා.
- **Rows (පේළි)**: එක පේළියක් කියන්නේ එක ළමයෙක්ගේ සම්පූර්ණ විස්තරයක්.

---

### 2. අපේ Project එකේ තියෙන ප්‍රධාන Tables (වගු) මොනවද?

අපේ Project එකේ ඔක්කොම වගු 15ක් විතර තියෙනවා. ඒත් මුලින්ම මේ 3 දැනගත්තම ඇති:

1. **`users` Table එක**: 
   - site එකේ register වෙලා ඉන්න හැමෝගෙම විස්තර තියෙන්නේ මෙතන. 
   - *Columns:* `id`, `full_name`, `email`, `password`, `role` (ළමයෙක්ද, Admin ද කියලා).

2. **`events` Table එක**: 
   - විශ්වවිද්‍යාලයේ තියෙන උත්සව ගැන විස්තර තියෙන්නේ මෙතන.
   - *Columns:* `id`, `title` (නම), `date` (දිනය), `location` (ස්ථානය).

3. **`marketplace_items` Table එක**: 
   - ළමයි විකුණන්න දාලා තියෙන බඩු වල විස්තර.
   - *Columns:* `id`, `title`, `price`, `status` (විකිණිලා ද නැද්ද).

---

### 3. කෙනෙක් අලුතින් Register වෙනකොට Database එකේ වෙන දේ

හිතන්න නිලේෂ් කියලා කෙනෙක් අලුතින් site එකට ඇවිත් Sign Up වෙනවා කියලා.

1. නිලේෂ් එයාගේ නමයි, email එකයි, password එකයි ගහලා Sign Up ඔබනවා.
2. Backend (PHP) එක ඒ විස්තර ටික අරගෙන **`users` Table එකේ අලුත් පේළියක් (Row එකක්) ලියනවා.**

එතකොට Table එක පේන්නේ මෙහෙමයි:

| id | full_name | email              | password       | role    |
|----|-----------|--------------------|----------------|---------|
| 1  | උතුම්     | uthum@uwu.ac.lk   | hgf654dfdsf... | student |
| 2  | නිලේෂ්   | nilesh@uwu.ac.lk  | df8s7dfsdfd... | student |

*(Password එක අකුරු ගොඩක් විදිහට හැරවිලා තියෙන්නේ ආරක්ෂාවට. ඒකට කියන්නේ Hashing කියලා. Admin ටවත් password එක කියවන්න බෑ!)*

---

### 4. ඔයාට වෙනස්කම් කරන්න ඕනෙ නම් කොහොමද කරන්නේ?

ඔයාට කවුරුහරි කිව්වොත්: 
*"නිලේෂ් කියන ළමයාව 'Admin' කෙනෙක් කරන්න ඕනේ!"*

1. මේක Database එකේ තියෙන අගයක් වෙනස් කරන වැඩක්.
2. Database එක ඇතුළට ගිහින් (phpMyAdmin වගේ සොෆ්ට්වෙයා එකක් පාවිච්චි කරලා) **`users` Table** එක අරින්න.
3. ඒකේ නිලේෂ් ඉන්න පේළිය (Row එක) හොයාගන්න. (උදාහරණෙ හැටියට id එක 2).
4. එතන `role` කියන Column එක යටතේ තියෙන "student" කියන වචනේ මකලා "admin" (හෝ 'superadmin') කියලා Type කරලා සේව් කරන්න.
5. දැන් නිලේෂ් site එකට ලොග් වෙද්දී එයාට Admin panel එක පේනවා!

ඔයාට කවුරුහරි කිව්වොත්: 
*"Event එකක නමක් වැරදියි, ඒක හදන්න ඕනේ!"*
1. **`events` Table** එකට යන්න.
2. අදාළ Event එක තියෙන පේළිය හොයාගන්න.
3. `title` කියන Column එකේ තියෙන නම අලුත් කරලා save කරන්න.

---
**දැන් ඔයාට Database එකක් කියන්නේ මොකක්ද සහ ඒක ඇතුළේ දත්ත තියෙන්නේ කොහොමද කියලා සරල අදහසක් තියෙනවා.**

ඊළඟ (3 වෙනි) කොටසෙන් අපි කතා කරමු **Frontend (ඉස්සරහා පේන දේ)** හැදිලා තියෙන්නේ කොහොමද සහ පාටක්, වචනයක් වගේ දෙයක් වෙනස් කරන්නේ කොහොමද කියලා හරියටම බලමු!
