# 🎓 UWU-NEXUS සරල මාර්ගෝපදේශය - (Extend Guide)
## 4 වෙනි කොටස: Backend එක (පිටිපස්සේ වැඩ කරන මොළය)

අපි කතා කළානේ Frontend එක කියන්නේ පේන දේවල් කියලා. හැබැයි ලස්සනට පෙනුණට විතරක් මදි, ඒක ඇතුළේ වැඩ කෙරෙන්නත් එපැයි. ඒ වැඩ ටික කරන **"මොළය"** තමයි Backend එක.

---

### 1. Backend එක කරන්නේ මොකක්ද?

කලින් කිව්ව වගේ මේක හරියට **රෙස්ටුරන්ට් එකක කුස්සියක්** වගේ. 
පාරිභෝගිකයා (Frontend එක) ඉල්ලන දේ මොකක්ද කියලා තේරුම් අරගෙන, ගබඩාවට (Database එකට) ගිහින් අදාළ දේවල් අරගෙන, කෑම එක හදලා පාරිභෝගිකයාට දෙන එක තමයි Backend එක කරන්නේ.

Frontend එකට තනියම Database එකත් එක්ක කතා කරන්න බෑ (ඒක ආරක්ෂිත නෑ). ඒක නිසා හැමතිස්සෙම Backend එක මැද ඉඳන් ඒ වැඩේ කරලා දෙනවා.

---

### 2. Backend ෆයිල්ස් තියෙන්නේ කොහෙද?

අපේ project ෆෝල්ඩරය ඇතුළේ තියෙන **`backend` ෆෝල්ඩරය** ඇතුළේ තමයි මේ මොළය තියෙන්නේ.
මේවා හැදිලා තියෙන්නේ **PHP** කියන තාක්ෂණයෙන්.

උදාහරණ කිහිපයක්:
- `login.php` = කෙනෙක් ලොග් වෙනකොට වැඩ කරන ෆයිල් එක
- `get_events.php` = විශ්වවිද්‍යාලයේ උත්සව (Events) මොනවද කියලා හොයලා දෙන ෆයිල් එක
- `create_event.php` = අලුත් උත්සවයක් Database එකට ඇතුලත් කරන ෆයිල් එක

---

### 3. මේක වැඩ කරන්නේ කොහොමද? (උදාහරණයක් බලමු)

හිතන්න ළමයෙක් වෙබ් අඩවියේ **"Events"** කියන පිටුවට යනවා කියලා. එතකොට කොහොමද එතන උත්සව වල විස්තර පෙන්නන්නේ? 

මේ පියවර 4 න් තමයි ඒක වෙන්නේ:

1. **Frontend එකේ ඉල්ලීම**: `app/events/page.tsx` ෆයිල් එක Backend එකට කෝල් එකක් දෙනවා, *"මට Events ටික දෙන්න"* කියලා.
2. **Backend එකේ වැඩේ**: `backend/get_events.php` කියන ෆයිල් එක ඒ කෝල් එක ආන්සර් කරනවා. 
3. **Database එකෙන් ඇහීම**: Backend එක Database එකට ගිහින් අහනවා, *"events කියන වගුවේ (Table එකේ) තියෙන ඔක්කොම දේවල් මට දෙන්න"* කියලා.
4. **පිළිතුර යැවීම**: Database එකෙන් දෙන විස්තර ටික Backend එක විසින් ලස්සනට පාර්සල් කරලා Frontend එකට යවනවා. ඊටපස්සේ Frontend එක ඒක තිරයේ පෙන්නනවා.

---

### 4. ඔයාට වෙනස්කමක් කරන්න ඕනේ නම් කොහොමද කරන්නේ?

හිතන්න ඔයාට කවුරුහරි කිව්වොත්:
*"කෙනෙක් වැරදි පාස්වර්ඩ් එකක් ගහලා ලොග් වෙන්න හැදුවොත් එන Message එක 'Invalid Email or Password' කියලා තියෙනවා, ඒක 'ඔබගේ මුරපදය වැරදියි' කියලා සිංහලට වෙනස් කරන්න!"*

මේක කරන්නේ මෙහෙමයි:
1. මේක Login වෙනකොට වෙන දෙයක් නිසා, යන්න ඕනේ **`backend` ෆෝල්ඩරයට**.
2. ඒක ඇතුළේ තියෙන `login.php` කියන ෆයිල් එක Notepad වගේ එකකින් අරින්න.
3. ඒකේ ලියලා තියෙන කෝඩ් එක කියවන් යද්දී ඔයාට පෙනෙයි මෙන්න මේ වගේ පේළියක්:
   `"message" => "Invalid Email or Password"`
4. අන්න ඒකෙ තියෙන ඉංග්‍රීසි වචන ටික මකලා ඒ වෙනුවට `"message" => "ඔබගේ මුරපදය වැරදියි"` කියලා Type කරන්න.
5. ෆයිල් එක Save කරන්න. දැන් කවුරුහරි වැරදි පාස්වර්ඩ් එකක් ගැහුවොත් අර සිංහල Message එක පෙනෙයි!

---

### 5. ලොකුම වාසිය මොකක්ද?

Backend එක Frontend එකෙන් වෙන් කරලා තියෙන නිසා තියෙන ලොකුම වාසිය තමයි **ආරක්ෂාව**. 
කවුරුහරි වෙබ් අඩවියට ආවම එයාට පේන්නේ Frontend එක විතරයි. එයාට කවදාවත් Backend එකේ තියෙන PHP ෆයිල්ස් බලන්නවත්, Database එකේ තියෙන පාස්වර්ඩ් බලන්නවත් බෑ. මොකද ඒවා තියෙන්නේ වෙනමම ආරක්ෂිත තැනක!

---
**දැන් ඔයාට Frontend එක, Backend එක සහ Database එක කියන 3නම වැඩ කරන විදිහ ගැන සම්පූර්ණ අදහසක් තියෙනවා.**

ඊළඟ (5 වෙනි) කොටසෙන් අපි කතා කරමු **Events සහ Tickets (උත්සව සහ ටිකට් ගැනීම)** කියන දේ ඇත්තටම මේ 3න ඇතුළේ වැඩ කරන්නේ කොහොමද කියලා.
