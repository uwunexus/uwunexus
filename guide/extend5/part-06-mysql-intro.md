# Part 06: Database එකක් Excel Sheet එකක් වගේ තේරුම් ගනිමු (MySQL Intro)

අපි කලින් කතා කළා අපේ පද්ධතියේ "මතකය" හෙවත් දත්ත ගබඩාව විදිහට අපි පාවිච්චි කරන්නේ **MySQL** කියන Database එක කියලා. Database එකක් කිව්වම ගොඩක් අයට මේක හරිම සංකීර්ණ දෙයක් වගේ පේනවා. හැබැයි මේක හරිම සරලයි.

Database එකක් තේරුම් ගන්න තියෙන හොඳම ක්‍රමය තමයි ඒක **Microsoft Excel** (එක්සෙල්) ෆයිල් එකකට සමාන කරන එක. 

---

## MySQL සහ Excel අතර සමානකම්

ඔබ සාමාන්‍යයෙන් Excel ෆයිල් එකක් හදද්දී මොකද කරන්නේ? 
1. මුලින්ම අලුත් Excel ෆයිල් එකක් (Workbook එකක්) හදනවා.
2. ඒක ඇතුළේ වෙන වෙනම Sheets (පිටු) හදනවා (උදා: Sheet 1 - ළමයින්ගේ නම්, Sheet 2 - විභාග ලකුණු).
3. හැම Sheet එකකම උඩින්ම මාතෘකා (Headings) දානවා (උදා: නම, වයස, ලිපිනය).
4. ඊටපස්සේ ඒ මාතෘකා යටතේ පේළියෙන් පේළියට (Rows) විස්තර පුරවනවා.

MySQL Database එකක් කියන්නෙත් හරියටම මේ දේම තමයි! නම් විතරයි වෙනස් වෙන්නේ:

| Excel වලදී කියන නම | MySQL වලදී කියන නම (Database භාෂාව) |
|---|---|
| Excel File (Workbook) | **Database** (දත්ත ගබඩාව) |
| Sheet (පිටුව) | **Table** (වගුව) |
| Headings (මාතෘකා - A,B,C) | **Columns / Fields** (තීරු) |
| Rows (පේළි - 1,2,3) | **Rows / Records** (වාර්තා) |

## අපේ UWU-NEXUS Database එක 

අපේ Project එකේ Database එකේ නම තමයි `uwunexus`. මේක හරියට ලොකු Excel ෆයිල් එකක් වගේ. ඒක ඇතුළේ වෙනස් වැඩ වලට වෙනම Tables (Sheets) හැදිලා තියෙනවා.

උදාහරණ කිහිපයක්:
* **`users` Table එක:** මේකේ තියෙන්නේ අපේ කැම්පස් එකේ ළමයින්ගේ විස්තර (නම, ඊමේල් එක, පාස්වර්ඩ් එක).
* **`events` Table එක:** මේකේ තියෙන්නේ ඉදිරියට තියෙන උත්සව ගැන විස්තර.
* **`marketplace_items` Table එක:** මේකේ තියෙන්නේ ළමයි විකුණන්න දාලා තියෙන බඩු වල විස්තර.

මේ එක Table එකක් (උදාහරණයක් විදිහට `users` table එක) ඇතුළේ තියෙන Columns (මාතෘකා) මෙන්න මේ වගේ වෙන්න පුළුවන්:

| id | first_name | last_name | email | password |
|---|---|---|---|---|
| 1 | Nilesh | Jayanandana | nilesh@gmail.com | 123456 |
| 2 | Nimal | Perera | nimal@yahoo.com | abcdef |

## Primary Key (හැඳුනුම්පත) යනු කුමක්ද?

ඉහත වගුවේ මුලින්ම තියෙන **`id`** කියන Column එක ගොඩක් වැදගත්. 

හිතන්න අපේ කැම්පස් එකේ "නිමල් පෙරේරා" කියලා ළමයි දෙන්නෙක් ඉන්නවා කියලා. එතකොට කොහොමද අපි මේ දෙන්නා වෙන් කරලා අඳුරගන්නේ? අන්න ඒකට තමයි අපි කැම්පස් එකෙන් ළමයින්ට වෙනස් වෙන්නැති අංකයක් (Registration Number) දෙන්නේ.

Database වලදීත් හැම පේළියකටම (Record එකකටම) **කවදාවත් සමාන වෙන්නේ නැති, එකිනෙකට වෙනස් අංකයක්** දෙනවා. ඒකට තමයි **Primary Key (ප්‍රාථමික යතුර)** කියලා කියන්නේ. ගොඩක් වෙලාවට මේකට අපි පාවිච්චි කරන්නේ `id` කියන Column එකයි. අංක 1 ආයේ කවදාවත් වෙන කෙනෙක්ට හම්බවෙන්නේ නැහැ.

> **සාරාංශය:**
> * Database එකක් කියන්නේ Excel ෆයිල් එකක් වගේ දෙයක්.
> * ඒක ඇතුළේ වෙනස් ජාතියේ දත්ත දාන්න වෙනම Tables තියෙනවා.
> * හැම Table එකකම අනිවාර්යයෙන්ම හැම වාර්තාවක්ම වෙන් කරලා අඳුරගන්න පුළුවන් Primary Key (`id`) එකක් තියෙනවා.

දැන් අපි Database එකක් කියන්නේ මොකක්ද කියලා හරියටම දන්නවා. ඊළඟ පාඩමෙන් අපි බලමු, අපේ කෝකියා (PHP) කොහොමද මේ ගබඩාවට (MySQL) කතා කරලා දත්ත ඉල්ලගන්නේ සහ අලුතින් දත්ත ඇතුළත් කරන්නේ කියලා! (SQL Queries).
