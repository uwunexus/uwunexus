# Part 09: UWU-NEXUS හි OOP භාවිතය 

කලින් පාඩමෙන් අපි OOP කියන්නේ මොකක්ද කියලා (Class එකක් සහ Object එකක් කියන්නේ මොකක්ද කියලා) ඉගෙන ගත්තා. දැන් අපි බලමු අපේ UWU-NEXUS Project එකේ ඇත්තටම මේ දේවල් පාවිච්චි වෙන්නේ කොහොමද කියලා.

ඔයා අපේ Project එකේ Backend එකේ තියෙන PHP ෆයිල් අරන් බැලුවොත්, ඔයාට අපි ලියපු `class User { ... }` වගේ දේවල් දකින්න හම්බවෙන්නේ නැහැ. එහෙනම් මේකේ OOP නැද්ද? 

නැහැ, මේකෙත් OOP තියෙනවා! හැබැයි අපි අලුතින් Classes ලියනවා වෙනුවට කරලා තියෙන්නේ, **PHP භාෂාව ඇතුළෙම කලින් හදලා තියෙන (Built-in) Classes පාවිච්චි කරන එකයි.**

---

## 1. Database එකට සම්බන්ධ වීම (PDO Class එක)

අපේ Project එකේ `backend/db.php` කියන ෆයිල් එක තමයි PHP මොළය සහ MySQL ගබඩාව අතර පාලම හදන්නේ. ඒ ෆයිල් එක ඇතුළේ මෙන්න මේ වගේ කෝඩ් එකක් තියෙනවා:

```php
<?php
// ... කලින් ලියපු විස්තර ...

$dsn = "mysql:host=$host;dbname=$db;charset=$charset";

try {
    // මෙන්න මෙතන තමයි OOP තියෙන්නේ!
    $pdo = new PDO($dsn, $user, $pass, $options);
    
} catch (\PDOException $e) {
    // මෙතනත් තව OOP Class එකක් තියෙනවා
    echo "Database connection failed";
}
?>
```

මෙතනදී වෙලා තියෙන දේ බලමු:
1. **`PDO` කියන්නේ Class එකක්:** මේක PHP හදපු අය විසින් ලියලා තියෙන "සැලැස්මක්". මේ සැලැස්ම ඇතුළේ ලියලා තියෙනවා Database එකක් එක්ක කතා කරන්නේ කොහොමද කියලා. අපි ආයෙ මුල ඉඳන් ඒක ලියන්න ඕනේ නැහැ.
2. **`new PDO(...)`:** අපි අර කලින් ඉගෙන ගත්තා වගේම `new` කියන වචනේ පාවිච්චි කරලා ඒ Class (සැලැස්ම) එකෙන් ඇත්තම Object එකක් හදනවා.
3. **`$pdo`:** ඒ හදපු Object එක අපි මේ පෙට්ටියට දාගන්නවා. දැන් අපිට Database එකෙන් මොනවා හරි ඉල්ලන්න ඕනේ නම් අපි කතා කරන්නේ මේ `$pdo` ටයි.

## 2. වැරදි අල්ලගැනීම (PDOException Class එක)

සමහර වෙලාවට Database එකට සම්බන්ධ වෙද්දී මොනවා හරි අවුලක් යන්න පුළුවන් (උදා: පාස්වර්ඩ් එක වැරදියි). එහෙම වුණොත් PHP එකෙන් අපිට එවනවා "Error" එකක්. 

මේ Error එක නිකන්ම වචනයක් නෙවෙයි. ඒකත් Object එකක්!
`catch (\PDOException $e)` කියන තැනින් කරන්නේ, එහෙම Error එකක් ආවොත් ඒක අල්ලගෙන `$e` කියන පෙට්ටියට දාගන්න එකයි. මේ `PDOException` කියන්නෙත් PHP වල තියෙන තවත් එක Class (සැලැස්මක්).

## Modern Web Development වල රහස: Paradigms මික්ස් කිරීම!

ඔයාට දැන් හිතෙන්න පුළුවන් "ඇයි අපිම අලුතින් Classes හැදුවේ නැත්තේ?" කියලා.

අද කාලේ (Modern) සොෆ්ට්වෙයාර් හදද්දී, එකම විදිහකට හැමදේම කරන්නේ නැහැ. වැඩේ ලේසි වෙන විදිහට ක්‍රම (Paradigms) කීපයක් එකතු කරනවා.
* **Backend (PHP) එකේදී:** Database එකට කතා කරන්න වගේ සංකීර්ණ දේවල් වලට අපි `PDO` වගේ **OOP (Objects)** පාවිච්චි කරනවා. අනිත් සාමාන්‍ය වැඩ වලට අපි සරල Functions පාවිච්චි කරනවා.
* **Frontend (Next.js) එකේදී:** ඔයා Next.js කෝඩ් එක බැලුවොත් ඒකේ Classes ඇත්තෙම නැහැ! ඒකෙ පාවිච්චි කරන්නේ සම්පූර්ණයෙන්ම **Functional Programming** කියන අනිත් රටාවයි. ඒකෙදී හැමදේම ලියන්නේ Functions විදිහටයි.

> **සාරාංශය:**
> * අපේ UWU-NEXUS එකේ අලුතින් ලියපු Classes නැහැ.
> * හැබැයි Database එකට Connect වෙන්න අපි PHP වල තියෙන **`PDO`** කියන Class එක පාවිච්චි කරලා Object එකක් හදාගන්නවා (`new PDO()`).
> * අද කාලේ ගොඩක් වෙලාවට OOP ක්‍රමයයි, Functional ක්‍රමයයි දෙකම කලවම් කරලා තමයි ලොකු Projects හදන්නේ.

මෙතනින් අපේ OOP පාඩම ඉවරයි! ඊළඟ පාඩමෙන් අපි අපේ **Module 4** ආරම්භ කරනවා. ඒ කියන්නේ ඇත්තටම අපේ UWU-NEXUS Project එකේ ෆෝල්ඩර ඇතුළට ගිහින් (File Structure එක) මොනවද මේවගේ තියෙන්නේ කියලා එකින් එක බලන්නයි යන්නේ.
